🔍 SHAP & Feature Importance — Attribute a Prediction to Each Feature

The model approved the loan. SHAP tells you which facts earned the yes — and by exactly how much.

Below is a small, transparent loan-approval model over five features. Pick an applicant and it returns an approval probability — but the interesting question is why. SHAP borrows the Shapley value from cooperative game theory: treat the features as players cooperating to produce the prediction, and hand each one a fair share of the payout — the amount the prediction sits away from the model's average output. Each feature's share is its average marginal contribution across every coalition of the other features, and here it is computed exactly — all 25 = 32 subsets enumerated, no approximation. The waterfall starts at the base value (an average applicant) and each feature pushes the probability up (red) or down (blue) until it lands on this applicant's number. Because the shares are fair, they add up exactly to prediction − base. Drag any feature and re-explain; the bar chart on the right ranks features by their global impact across many applicants.

Local (the waterfall) explains one row; global averages the magnitude of each feature's SHAP over many rows to rank what the model relies on overall.
Why the shares always reconcile: the Shapley value obeys an efficiency axiom — for any way of defining a "missing" feature, the exact shares sum to v(all) − v(∅), i.e. prediction − base value. So the waterfall is a closed accounting, not a heuristic: every point of the gap between the average applicant and this one is handed to some feature. The catch is cost — the exact sum runs over 2n coalitions, fine for 5 features (32) but hopeless for 100. That is why production libraries approximate: KernelSHAP samples coalitions and fits a weighted regression, while TreeSHAP exploits tree structure to get the exact values in polynomial time — the reason SHAP pairs so naturally with XGBoost and LightGBM.

🔨 Build it yourself

    
Define a small transparent model and a background, spell out the value function v(S) that marginalises the "absent" features, then compute exact Shapley values by enumerating every coalition. Check that they sum to prediction − base (the efficiency axiom), aggregate them into global importance — and finally reach for the real shap library with TreeExplainer.

    
    
      
  1. 
        
    1
    The model & the background
    
        
    SHAP explains predictions relative to a baseline. That baseline is a background distribution — here just the population means. The base value is the model's output on that background: the prediction for a featureless "average" applicant.
    
        
    // a small, transparent logistic model: loan-approval probability
const W = { income:0.02, credit:0.01, debt:-0.05, age:0.01, emp:0.05 };
const B = -7.4;
const MEAN = { income:60, credit:650, debt:20, age:40, emp:6 };   // background
const sigmoid = z => 1/(1+Math.exp(-z));
const FEATS = ["income","credit","debt","age","emp"];
const model = x => sigmoid(B + FEATS.reduce((z,f) => z + W[f]*x[f], 0));

const base = model(MEAN);   // v(∅): average applicant → 0.500

    2
    The value function v(S)
    
        
    A game needs a payout for every coalition — every subset of features acting together. But the model needs all its inputs, so we define "present" features (in S) at the applicant's values and marginalise the rest to the background. That is what a "missing" feature means.
    
        
    function blend(x, S){                    // S = Set of "present" feature names
  const o = {};
  for (const f of FEATS) o[f] = S.has(f) ? x[f] : MEAN[f];
  return o;                              // known features kept, rest → background
}
const v = (x, S) => model(blend(x, S));   // expected output knowing only S
// v(x, ∅) === base   ·   v(x, all features) === model(x)

    Exact Shapley by enumeration
    
        
    For each feature, sum its marginal contribution v(S∪{i})−v(S) over every subset S of the other features, weighted so the sum equals the average over all n! orderings. With n=5 that is 24=16 coalitions per feature — exact, no sampling.
    
        
    const fact = k => { let r=1; for (let i=2;i<=k;i++) r*=i; return r; };
const n = FEATS.length;

function shap(x){
  const phi = {};
  for (let i=0;i<n;i++){
    const fi = FEATS[i], others = FEATS.filter((_,j) => j!==i);
    let acc = 0;
    for (let m=0; m<(1<<others.length); m++){          // every subset S of the others
      const S = new Set(others.filter((_,b) => m & (1<<b)));
      const w = fact(S.size)*fact(n-S.size-1)/fact(n);  // |S|!(n-|S|-1)!/n!
      acc += w * (v(x, new Set([...S, fi])) - v(x, S)); // marginal contribution
    }
    phi[fi] = acc;                                      // Shapley value of feature fi
  }
  return phi;
}

    Efficiency check & the waterfall
    
        
    The defining property: the SHAP values sum exactly to prediction − base. Nothing is created or lost. The waterfall starts at the base and adds each φ (biggest first) until it lands on f(x) — red when it pushes up, blue when it pushes down.
    
        
    const x = { income:90, credit:720, debt:35, age:33, emp:4 };
const phi = shap(x);
const sum = FEATS.reduce((s,f) => s + phi[f], 0);

console.assert(Math.abs(sum - (model(x) - base)) < 1e-9, "efficiency"); // always

let run = base;                                     // build the waterfall
FEATS.slice().sort((a,b) => Math.abs(phi[b]) - Math.abs(phi[a]))
     .forEach(f => { const from = run; run += phi[f]; /* red if phi>0 else blue */ });
// run === model(x)  — the bars close the gap exactly

    Local → global importance
    
        
    One explanation is local. Average the absolute SHAP value of each feature over many rows and you get global importance — how much the feature swings predictions overall. Absolute, so up-pushes and down-pushes don't cancel.
    
        
    const globalImp = {};
FEATS.forEach(f => globalImp[f] = 0);
for (const r of DATA){                    // your dataset of applicants
  const p = shap(r);
  FEATS.forEach(f => globalImp[f] += Math.abs(p[f]));
}
FEATS.forEach(f => globalImp[f] /= DATA.length);   // mean |SHAP| per feature
// rank features by globalImp -> the summary bar chart

    In the real world — the shap library
    
        
    Enumerating 2n coalitions dies past a handful of features. shap gives you TreeExplainer — exact SHAP for tree ensembles in polynomial time — plus the waterfall, force and beeswarm plots, and a model-agnostic KernelExplainer fallback.
    
        
    # pip install shap
import shap, xgboost

model = xgboost.XGBClassifier().fit(X, y)      # any tree model
explainer = shap.TreeExplainer(model)          # EXACT SHAP for trees, fast
sv = explainer(X)                              # SHAP values for every row

shap.plots.waterfall(sv[0])                    # explain ONE prediction (local)
shap.plots.force(sv[0])                        # same numbers, force layout
shap.plots.bar(sv)                             # mean |SHAP| -> global importance
shap.plots.beeswarm(sv)                        # per-feature spread across all rows

# model-agnostic fallback (any model, sampled coalitions):
