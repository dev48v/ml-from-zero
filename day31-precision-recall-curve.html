🎯 Precision-Recall Curve — The Metric That Survives a Rare Positive Class

Your fraud model reports 0.98 ROC-AUC. Then you deploy it and 95% of its alerts are false alarms. The precision-recall curve is the plot that would have warned you.

A classifier does not output a class — it outputs a score (a probability). To turn scores into decisions you pick a threshold. Slide the threshold and every count in the confusion matrix moves, so precision and recall move too. A precision-recall curve traces that trade-off across every threshold: recall on the x-axis, precision on the y-axis. Everything below is computed for real, live in your browser — no pre-baked images. We draw positive and negative examples from two overlapping score distributions, sweep the threshold from 1→0, and plot the curve, the moving operating point, the live confusion matrix, and the Average Precision (area under the PR curve). Beside it we plot the same data's ROC curve. Now crank the class-imbalance slider and watch the lesson land: ROC stays rosy while PR collapses toward the prevalence floor.

Precision = TP / (TP+FP): of everything you flagged, how much was right. Recall = TP / (TP+FN): of everything that was positive, how much you caught. Lower the threshold → catch more (recall ↑) but flag more junk (precision ↓).

🔨 Build it yourself

    
Write the threshold sweep from scratch — sort by score, walk down, accumulate TP and FP — then reach for sklearn.metrics.precision_recall_curve and average_precision_score, which do exactly this. The point is to stop trusting a single accuracy or ROC number when the positive class is rare, and read the curve that actually predicts how the model behaves in production.

    
    
      
  1. 
        
    1
    The threshold sweep from scratch
    
        
    Sort examples by score, high to low. Walk down the list: each new example you "include" is a prediction of positive. Keep running tp and fp counts — that is enough to read precision and recall at every threshold without ever re-scanning the data.
    
        
    // data: [{score, y}], y in {0,1}
function prCurve(data, nPos){
  const sorted = [...data].sort((a,b)=>b.score-a.score);
  let tp=0, fp=0;
  const pts=[{recall:0, precision:1}];   // threshold above every score
  for (const e of sorted){
    if (e.y===1) tp++; else fp++;
    pts.push({
      recall:    tp/nPos,                // = TP/(TP+FN)
      precision: tp/(tp+fp)              // = TP/(TP+FP)
    });
  }
  return pts;                            // recall runs 0 -> 1
}

    Precision, recall, F1 at one threshold
    
        
    A single operating point is just the confusion matrix at a fixed threshold t. Note recall's denominator is the positive count only — it never sees a true negative — which is the seed of the whole PR-vs-ROC story.
    
        
    function metricsAt(data, t){
  let tp=0, fp=0, fn=0, tn=0;
  for (const e of data){
    const pred = e.score >= t ? 1 : 0;
    if (e.y===1 && pred===1) tp++;
    else if (e.y===0 && pred===1) fp++;
    else if (e.y===1 && pred===0) fn++;
    else tn++;
  }
  const precision = tp+fp ? tp/(tp+fp) : 1;   // guarded /0
  const recall    = tp+fn ? tp/(tp+fn) : 0;
  const f1 = precision+recall
    ? 2*precision*recall/(precision+recall) : 0;
  return {tp,fp,fn,tn, precision, recall, f1};
}

    Average Precision — area under the PR curve
    
        
    AP is a weighted sum: at each threshold, weight the precision by the jump in recall since the previous point. Equivalently — because recall only jumps when you hit a true positive — AP is the mean precision measured at every positive. That is exactly what scikit-learn's average_precision_score computes (no interpolation).
    
        
    function averagePrecision(pts){          // pts from prCurve()
  let ap=0;
  for (let i=1;i<pts.length;i++){
    ap += (pts[i].recall - pts[i-1].recall) * pts[i].precision;
  }
  return ap;                             // in [0,1]
}
// identical to: mean( precision at each positive ),
// which is why AP rewards ranking positives near the top.

    scikit-learn: the two functions you actually call
    
        
    In practice you never hand-roll the sweep. Feed scores (not hard labels) to precision_recall_curve and average_precision_score. The curve returns one extra point (precision=1, recall=0) and thresholds is one shorter than the arrays.
    
        
    import numpy as np
from sklearn.metrics import precision_recall_curve, average_precision_score

y_score = model.predict_proba(X_test)[:, 1]   # scores, NOT .predict()

precision, recall, thresholds = precision_recall_curve(y_test, y_score)
ap = average_precision_score(y_test, y_score)  # area under PR

baseline = y_test.mean()          # no-skill line = positive prevalence
print(f"AP={ap:.3f}   baseline={baseline:.3f}")

    Plot it — with the baseline and both curves
    
        
    Always draw the no-skill baseline at the prevalence, and plot ROC beside PR so the imbalance gap is visible. PrecisionRecallDisplay does the plotting; add the baseline yourself.
    
        
    import matplotlib.pyplot as plt
from sklearn.metrics import PrecisionRecallDisplay, RocCurveDisplay

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(11,4))
PrecisionRecallDisplay.from_predictions(y_test, y_score, ax=ax1)
ax1.axhline(y_test.mean(), ls="--", c="grey", label="no-skill")  # prevalence
ax1.legend()

RocCurveDisplay.from_predictions(y_test, y_score, ax=ax2)         # contrast
# On imbalanced data ROC-AUC barely moves while AP collapses.

    Pick an operating point by cost
    
        
    The curve summarises the model; deployment needs one threshold. Choose it from the business cost, not 0.5. Two common recipes: maximise F1 (balanced), or take the highest recall that still clears a precision floor (e.g. "alerts must be right >=80% of the time").
    
        
    import numpy as np
from sklearn.metrics import precision_recall_curve

precision, recall, thr = precision_recall_curve(y_test, y_score)

# recipe A: best F1
f1 = 2*precision*recall / (precision+recall + 1e-12)
t_f1 = thr[np.nanargmax(f1[:-1])]           # last point has no threshold

# recipe B: highest recall with precision >= 0.80
ok = precision[:-1] >= 0.80
t_floor = thr[ok][np.argmax(recall[:-1][ok])] if ok.any() else 1.0

deploy = (y_score >= t_f1).astype(int)
