🎬 Collaborative Filtering — Recommend What People Like You Already Loved

You never told it what a movie is about. It recommends anyway.

Below is a tiny ratings matrix: 6 users × 6 movies, scored 1–5, with blanks where nobody watched. Collaborative filtering fills those blanks using one idea — people who agreed in the past will agree again. Pick a user; the demo finds their nearest neighbours by cosine similarity on the movies they both rated (mean-centred, so it captures taste, not just generosity), then predicts each blank as a similarity-weighted average and surfaces the top recommendation. Flip to item–item (compare movie columns instead — more stable) or to matrix factorization, which learns 2 hidden taste factors per user and per movie by SGD and reconstructs the whole grid. No genres, no descriptions — the taste structure falls out of the ratings alone. Click a filled cell to change a rating and watch the recommendations move.

Recommend for:
Collaborative vs content-based filtering: a content-based recommender needs features about each item (genre, cast, keywords) and recommends more of what you already liked — it can never surprise you outside your known tastes, and it needs someone to tag every item. Collaborative filtering uses no item features at all: it learns purely from the pattern of who rated what, so it can recommend a film in a genre you have never touched simply because people with your taste loved it. The price is the cold-start problem — a brand-new user or a brand-new movie has no ratings to lean on, so real systems blend CF with content-based and popularity fallbacks.

🔨 Build it yourself

    
A sparse ratings matrix, a mean-centred cosine, a similarity-weighted prediction, an item–item variant, a matrix factorizer trained by SGD, and an honest hold-out test — all in vanilla JS. Then the real library version in a handful of lines.

    
    
      
  1. 
        
    1
    The sparse ratings matrix
    
        
    Everything starts from one array: rows are users, columns are items, cells are explicit ratings, and null marks the vast majority you have never observed. A real matrix is 95–99% empty — the whole job of collaborative filtering is to fill those nulls.
    
        
    // rows = users, cols = movies, null = not rated yet
const R = [
  [5, null, 2, null, 3, 3],   // Ava
  [5, 5,    1, 2,    null, 4], // Ben
  // ...
];
const N = R.length, M = R[0].length;
const clamp = x => Math.max(1, Math.min(5, x));   // keep on the 1..5 scale
    
      
    2. 
      
  2. 
        
    2
    Similarity: mean-centred cosine
    
        
    Compare two users only on the items they both rated. Subtract each user's own mean first: this removes the "everything's a 4" easy raters from the "everything's a 2" harsh raters so the cosine measures the shape of taste. Mean-centred cosine on co-rated items is exactly the Pearson correlation, and it can go negative — opposite tastes.
    
        
    function simUsers(a, b){                       // Pearson over co-rated items
  let dot=0, na=0, nb=0, cnt=0;
  for (let i=0;i<M;i++){
    const x=R[a][i], y=R[b][i];
    if (x!=null && y!=null){
      const dx=x-uMean[a], dy=y-uMean[b];       // mean-centre both users
      dot+=dx*dy; na+=dx*dx; nb+=dy*dy; cnt++;
    }
  }
  if (cnt<2 || na===0 || nb===0) return 0;      // too little overlap
  return dot / Math.sqrt(na*nb);                // in [-1, 1]
}
    
      
    3. 
      
  3. 
        
    3
    User–user prediction: similarity-weighted average
    
        
    To predict user u's rating of item i, take the top-k neighbours who actually rated i, and average their deviations from their own means, weighted by similarity. Add back u's mean. Weighting by similarity means close neighbours dominate; centring by each neighbour's mean keeps a harsh rater's 4 from drowning a generous rater's 4.
    
        
    function predUser(u, i, k){
  const cand = [];
  for (let v=0; v<N; v++)
    if (v!==u && R[v][i]!=null){ const s=simUsers(u,v); if (s) cand.push({v,s}); }
  cand.sort((a,b)=>Math.abs(b.s)-Math.abs(a.s));       // strongest ties first
  const nb = cand.slice(0, k);                          // top-k neighbours
  let num=0, den=0;
  for (const {v,s} of nb){ num += s*(R[v][i]-uMean[v]); den += Math.abs(s); }
  return den===0 ? uMean[u] : clamp(uMean[u] + num/den);
}
    
      
    4. 
      
  4. 
        
    4
    Item–item CF (the stable variant)
    
        
    Flip the matrix: compare columns. Two movies are similar if the same people rated them alike — using adjusted cosine, which centres each rating by that user's mean. Item–item is what Amazon shipped ("customers who bought this…"): item-to-item tastes change far more slowly than user tastes, so you can precompute the item–item similarities offline and serve recommendations instantly.
    
        
    function simItems(i, j){                        // adjusted cosine over users
  let dot=0, ni=0, nj=0, cnt=0;
  for (let u=0;u<N;u++){
    const x=R[u][i], y=R[u][j];
    if (x!=null && y!=null){
      const dx=x-uMean[u], dy=y-uMean[u];        // centre by the USER's mean
      dot+=dx*dy; ni+=dx*dx; nj+=dy*dy; cnt++;
    }
  }
  return (cnt<2||ni===0||nj===0) ? 0 : dot/Math.sqrt(ni*nj);
}
function predItem(u, i, k){                      // weighted over items u rated
  const cand=[];
  for (let j=0;j<M;j++)
    if (j!==i && R[u][j]!=null){ const s=simItems(i,j); if (s) cand.push({j,s}); }
  cand.sort((a,b)=>Math.abs(b.s)-Math.abs(a.s));
  let num=0, den=0;
  for (const {j,s} of cand.slice(0,k)){ num+=s*(R[u][j]-iMean[j]); den+=Math.abs(s); }
  return den===0 ? iMean[i] : clamp(iMean[i] + num/den);
}
    
      
    5. 
      
  5. 
        
    5
    Matrix factorization by SGD (latent factors)
    
        
    Instead of comparing rows or columns, learn a small vector of hidden factors for every user and every movie so that their dot product reproduces the known ratings: R ≈ μ + P·Qᵀ. Stochastic gradient descent walks each known rating, computes the error, and nudges both factor vectors toward it (with a little L2 regularization). This is the family that won the Netflix Prize; the factors end up meaning things like "sci-fi ↔ romance."
    
        
    const K=2, lr=0.03, reg=0.05;                   // 2 latent factors
let P = randMatrix(N, K), Q = randMatrix(M, K); // small random init
const predMF = (u,i) => gMean + dot(P[u], Q[i]);

for (let epoch=0; epoch<300; epoch++)
  for (let u=0;u<N;u++) for (let i=0;i<M;i++){
    if (R[u][i]==null) continue;
    const err = R[u][i] - predMF(u,i);
    for (let f=0; f<K; f++){
      const pf=P[u][f], qf=Q[i][f];
      P[u][f] = pf + lr*(err*qf - reg*pf);       // gradient step on user factor
      Q[i][f] = qf + lr*(err*pf - reg*qf);       // and on item factor
    }
  }

    Evaluate: hold out ratings, beat the baseline
    
        
    A recommender is only good if it beats the dumb baseline of "predict the global average for everyone." Hide a slice of the known ratings, predict them, and score RMSE against the truth. If your CF or MF RMSE is not clearly below the global-average RMSE, something is wrong — this is the sanity check every recommender needs before it ships.
    
        
    function rmse(pred){
  let se=0, n=0;
  for (const [u,i,truth] of testSet){ const e = truth - pred(u,i); se += e*e; n++; }
  return Math.sqrt(se/n);
}
const baseline = rmse((u,i) => gMean);           // predict the global average
const cf       = rmse((u,i) => predUser(u,i,3)); // must be clearly lower
console.log(baseline, cf, cf < baseline);        // true = the model earns its keep

    In the real world
    
        
    Nobody hand-rolls this at scale. For explicit ratings, scikit-surprise gives you tuned KNN and SVD in a few lines. For implicit feedback (clicks, plays, buys — no stars), reach for implicit (weighted-ALS, BPR) or LightFM (hybrid CF + content, which softens cold-start). These handle sparse matrices, negative sampling, and evaluation for you.
    
        
    # pip install scikit-surprise implicit lightfm

# --- explicit ratings: matrix factorization with Surprise ---
from surprise import SVD, Dataset, Reader
from surprise.model_selection import cross_validate
data = Dataset.load_from_df(ratings[['user','item','rating']], Reader(rating_scale=(1,5)))
cross_validate(SVD(n_factors=50), data, measures=['RMSE'], cv=5, verbose=True)

# --- implicit feedback (plays/clicks): weighted-ALS ---
from implicit.als import AlternatingLeastSquares
model = AlternatingLeastSquares(factors=64, regularization=0.05)
model.fit(user_item_confidence)          # sparse matrix of confidences
recs = model.recommend(user_id, user_item_confidence[user_id], N=10)
