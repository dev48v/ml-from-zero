🌳 Hierarchical Clustering — Merge Nearest, Read the Dendrogram

Give every point its own cluster, then keep merging the two closest.

No k up front. Agglomerative hierarchical clustering starts with every point alone, then — over and over — finds the two closest clusters and fuses them, recording the distance at which they joined. Do that until a single cluster remains and you have a whole tree of nested groupings, drawn as a dendrogram on the right: the higher a horizontal bar, the more dissimilar the two groups it joins. Drag the cut line (or press Step / Run) to slice the tree at any height — everything below a cut becomes one flat cluster, and the scatter on the left recolours to match. Both views are the same computation, seen two ways.

This is a genuine agglomerative clustering, run live in your browser. Each redraw rebuilds the flat clusters by union-finding every merge whose height is at or below the cut line. The merge tree is computed by the real algorithm: start with 24 singletons, compute the linkage distance between every pair of current clusters, fuse the closest pair, record its height, and repeat 23 times until one cluster remains. single uses the minimum distance between members (prone to "chaining" — it can string dumbbell shapes together), complete uses the maximum (tight, compact balls), average uses the mean of all cross-pair distances, and Ward merges the pair that increases the total within-cluster variance the least (usually the cleanest blobs). Because the cut simply reads groups off the finished tree, you pick the number of clusters after seeing the whole structure — the big vertical gaps in the dendrogram are the natural places to cut.

vs. K-Means (Day 13): K-Means makes you commit to k before you look and only finds round, blob-shaped clusters. Hierarchical clustering asks for no k — it builds the entire nesting once, and you cut wherever the structure tells you to. With single linkage it can even trace non-convex shapes (chains, rings) that K-Means can never separate. The price is speed: it is O(n²) memory and up to O(n³) time, so it shines on hundreds-to-thousands of points, not millions.

🔨 Build it yourself

    
A distance matrix, a linkage rule, a greedy merge loop that records every join, and a cut that reads flat clusters off the tree. ~70 lines of vanilla JS — then the real-world SciPy version in a handful of lines.

    
    
      
  1. 
        
    1
    Start: every point is its own cluster
    
        
    Agglomerative clustering is bottom-up. With n points you begin with n clusters of size one and a full pairwise distance matrix. Euclidean distance is the usual base metric, but any metric works — cosine for text, correlation for gene expression. Everything downstream is just "which two clusters are closest, and by how much."
    
        
    function dist(a, b){                 // base Euclidean metric
  return Math.hypot(a[0]-b[0], a[1]-b[1]);
}
let clusters = points.map((p, i) => ({ members: [i] }));  // n singletons
// D[i][j] = pairwise distances between the original points
const D = points.map(a => points.map(b => dist(a, b)));
    
      
    2. 
      
  2. 
        
    2
    Define cluster-to-cluster distance (linkage)
    
        
    Point distance is obvious; cluster distance is a choice called the linkage. single takes the closest pair of members (chaining), complete the farthest pair (compact balls), average the mean over all cross-pairs, and Ward the increase in within-cluster variance if you merged them. The linkage completely changes the shape of the tree and the clusters you get.
    
        
    function linkDist(A, B, kind){
  if (kind === "ward"){                       // variance increase
    const ca = centroid(A), cb = centroid(B);
    const na = A.members.length, nb = B.members.length;
    const d2 = (ca[0]-cb[0])**2 + (ca[1]-cb[1])**2;
    return Math.sqrt((2*na*nb/(na+nb)) * d2);
  }
  const vals = [];
  for (const i of A.members) for (const j of B.members) vals.push(D[i][j]);
  if (kind === "single")   return Math.min(...vals);
  if (kind === "complete") return Math.max(...vals);
  return vals.reduce((s,v)=>s+v,0) / vals.length;   // average
}

  3. 
        
    3
    The merge loop: fuse the closest pair, record the height
    
        
    This is the whole algorithm. While more than one cluster remains, scan every pair of current clusters, find the minimum linkage distance, merge those two into one, and push a record of the join and its height onto the merge list. After n−1 merges you have a single cluster and a complete tree. The recorded heights are what make the dendrogram.
    
        
    const merges = [];
while (clusters.length > 1){
  let best = Infinity, bi = -1, bj = -1;
  for (let i=0;i<clusters.length;i++)
    for (let j=i+1;j<clusters.length;j++){
      const d = linkDist(clusters[i], clusters[j], kind);
      if (d < best){ best = d; bi = i; bj = j; }
    }
  const merged = { members: [...clusters[bi].members, ...clusters[bj].members] };
  merges.push({ a: bi, b: bj, height: best, size: merged.members.length });
  clusters = clusters.filter((_,k)=> k!==bi && k!==bj);
  clusters.push(merged);                       // n-1 merges total
}

  4. 
        
    4
    Cut the tree at a height → flat clusters
    
        
    The tree is the model; a flat clustering is just a horizontal slice of it. Apply every merge whose height is at or below your cut with a union-find, and whatever ends up connected is one cluster. Cut low and you get many tiny clusters; cut high and they collapse into a few big ones. The best cut is usually the tallest vertical gap — the biggest jump in merge height.
    
        
    function cutAt(merges, n, h){
  const parent = [...Array(n).keys()];
  const find = x => { while(parent[x]!==x){ parent[x]=parent[parent[x]]; x=parent[x]; } return x; };
  for (const m of merges)                       // apply merges up to height h
    if (m.height <= h)
      for (const p of m.members.slice(1)) parent[find(p)] = find(m.members[0]);
  const labels = {}; let next = 0, out = [];
  for (let i=0;i<n;i++){ const r=find(i); if(!(r in labels)) labels[r]=next++; out.push(labels[r]); }
  return out;                                    // one integer label per point
}

  5. 
        
    5
    Draw the dendrogram
    
        
    Lay the leaves left-to-right in the order a depth-first walk of the tree visits them, so branches never cross. Each internal node sits at the average x of its two children and at a y equal to its merge height. Draw a "U": a horizontal bar at the merge height joining the two children, with a vertical drop to each child. A single horizontal line at your cut height turns the picture into the flat clustering.
    
        
    // x of a node = mean of its children's x; y = merge height
function layout(node){
  if (node.leaf) return node.x;
  node.x = (layout(node.a) + layout(node.b)) / 2;
  return node.x;
}
function drawU(ctx, a, b, hgt){                 // one merge = one "U"
  ctx.beginPath();
  ctx.moveTo(a.x, yPix(a.h)); ctx.lineTo(a.x, yPix(hgt));   // left drop
  ctx.lineTo(b.x, yPix(hgt));                               // top bar
  ctx.lineTo(b.x, yPix(b.h)); ctx.stroke();                // right drop
}

    6
    In the real world
    
        
    Nobody hand-rolls this in production. SciPy's linkage builds the whole merge tree in one fast call, dendrogram plots it, and fcluster cuts it at a height or to a target number of clusters. Scikit-learn's AgglomerativeClustering gives the same thing with a familiar fit_predict. Scale your features first (linkage is distance-based), and remember it is O(n²) memory — for very large data reach for K-Means, DBSCAN, or BIRCH instead.
    
        
    # pip install scipy scikit-learn
from scipy.cluster.hierarchy import linkage, dendrogram, fcluster
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt

Xs = StandardScaler().fit_transform(X)
Z  = linkage(Xs, method="ward")        # the full merge tree (n-1 rows)

dendrogram(Z); plt.axhline(y=cut, c="r", ls="--"); plt.show()

labels = fcluster(Z, t=3, criterion="maxclust")   # cut into 3 clusters
# or: fcluster(Z, t=cut, criterion="distance")     # cut at a height

# sklearn equivalent:
from sklearn.cluster import AgglomerativeClustering
labels = AgglomerativeClustering(n_clusters=3, linkage="ward").fit_predict(Xs)
