🌐 UMAP — A Faster Map of High-Dim Data

Build a neighbour graph, then let it relax into a map.

Yesterday t-SNE squashed 6D blobs to 2D by matching pairwise probabilities and descending KL — accurate but slow, and it throws away global layout. UMAP takes a graph-first route. It connects every point to its k nearest neighbours in 6D, weights each edge by a smooth membership strength (a fuzzy simplicial complex), then lays the graph out in 2D by pulling connected points together and pushing everyone else apart — a force-directed layout that minimises the cross-entropy of the two fuzzy topologies. The dots start as one random blob; press Run and watch the four true clusters snap into separate islands, fast.

This is a genuine, if small, UMAP: a k-nearest-neighbour graph is built in 6D, each point's local connectivity (ρ = distance to its nearest neighbour) is subtracted and a per-point σ is binary-searched so the membership row sums to log₂(n_neighbors), directed edges are fused with the fuzzy union a+b−ab, the low-dim curve 1/(1+a·d²ᵇ) is least-squares fit from min_dist, and the 2D layout descends the cross-entropy by stochastic edge sampling with negative sampling — attract along edges, repel random pairs, learning rate decaying each epoch. Colours mark the true cluster so you can judge the result — UMAP never sees them. Separation (between-cluster ÷ within-cluster spread) should climb from <1 to well past 10. Small n_neighbors emphasises local detail; large values preserve more global shape. Larger min_dist packs each cluster less tightly.

vs. t-SNE (Day 23): UMAP is typically much faster (it optimises a sparse kNN graph with sampling, not a dense N² matrix), and it tends to preserve more global structure — the relative placement of clusters is more trustworthy than in t-SNE. The n_neighbors knob is the local↔global tradeoff: low means "believe only nearby points," high means "respect the broader manifold." UMAP can also project brand-new points onto an existing map, which t-SNE cannot.

🔨 Build it yourself

    
A kNN graph, per-point local connectivity, fuzzy-union edge weights, a min_dist-fit low-dim curve, and a sampled attract/repel layout loop. ~90 lines of vanilla JS — then the two-line real-world version.

    
    
      
  1. 
        
    1
    Find each point's k nearest neighbours
    
        
    UMAP is graph-first: it never touches most pairwise distances. For each point it keeps only the n_neighbors closest points in high-dim. Real implementations use approximate nearest-neighbour search (NN-Descent) for near-linear speed; here a small sort is enough. This sparse graph is why UMAP scales where t-SNE's dense N² matrix stalls.
    
        
    function knnGraph(X, k){
  const n = X.length;
  const knn = [];
  for (let i=0;i<n;i++){
    const order = [...Array(n).keys()]
      .filter(j => j!==i)
      .sort((a,b)=> dist(X[i],X[a]) - dist(X[i],X[b]));
    knn.push(order.slice(0, k));   // k closest points to i
  }
  return knn;
}

    2
    Local connectivity: ρ and σ
    
        
    For each point, ρ is the distance to its nearest neighbour — subtracting it guarantees every point is connected to at least one other with full weight, so nothing is orphaned. Then σ (a per-point bandwidth) is binary-searched so the smooth membership row sums to log₂(k). This adapts to local density exactly like t-SNE's perplexity, but the target is a graph, not a probability.
    
        
    function localSigma(dists, k){       // dists = sorted knn dists
  const rho = dists[0];              // nearest-neighbour distance
  const target = Math.log2(k);
  let lo=0, hi=Infinity, sig=1;
  for (let t=0;t<64;t++){
    let s=0;
    for (const d of dists){
      const v = Math.max(0, d - rho);
      s += v<=0 ? 1 : Math.exp(-v/sig);
    }
    if (Math.abs(s-target)<1e-5) break;
    if (s>target){ hi=sig; sig=(lo+hi)/2; }
    else { lo=sig; sig = hi===Infinity ? sig*2 : (lo+hi)/2; }
  }
  return { rho, sig };
}

  3. 
        
    3
    Fuzzy edge weights, then symmetrise
    
        
    Each directed edge i→j gets a membership strength exp(−(d−ρ)/σ) in (0,1] — a fuzzy "how much do they belong together." Because i's view of j differs from j's view of i, UMAP fuses them with the probabilistic t-conorm a + b − a·b (fuzzy union), giving one symmetric weight per pair. The result is the high-dim fuzzy graph — UMAP's answer to t-SNE's joint P.
    
        
    const w = Array.from({length:n},()=>new Float64Array(n));
for (let i=0;i<n;i++) for (const j of knn[i]){
  const v = Math.max(0, dist(X[i],X[j]) - rho[i]);
  w[i][j] = v<=0 ? 1 : Math.exp(-v / sig[i]);
}
const edges = [];
for (let i=0;i<n;i++) for (let j=i+1;j<n;j++){
  const a=w[i][j], b=w[j][i];
  const s = a + b - a*b;                 // fuzzy union
  if (s > 1e-3) edges.push({i, j, w:s});
}

  4. 
        
    4
    Fit the low-dim curve from min_dist
    
        
    In 2D, membership uses 1/(1 + a·d²ᵇ). The parameters a, b are least-squares fit so this curve matches a target that stays 1 up to min_dist, then decays. Small min_dist ⇒ a large a ⇒ points allowed to pack very tightly; large min_dist spreads clusters out. This one knob controls cluster tightness, the way t-SNE's Student-t kernel is fixed at a=b=1.
    
        
    function fitAB(min_dist){
  const xs=[], ys=[];
  for (let d=0; d<=3; d+=0.01){
    xs.push(d); ys.push(d<=min_dist ? 1 : Math.exp(-(d-min_dist)));
  }
  let a=1, b=1;
  for (let it=0; it<300; it++){                 // gradient descent
    let ga=0, gb=0;
    for (let k=0;k<xs.length;k++){
      const d=xs[k]; if (d===0) continue;
      const pw=Math.pow(d,2*b), den=1+a*pw, err=1/den - ys[k];
      ga += 2*err*(-pw/(den*den));
      gb += 2*err*(-a*pw*2*Math.log(d)/(den*den));
    }
    a -= 0.1*ga/xs.length; b -= 0.1*gb/xs.length;
  }
  return { a, b };
}

  5. 
        
    5
    Lay it out: attract, repel, decay
    
        
    Start with random 2D positions, then run stochastic gradient descent on the cross-entropy between the high-dim and low-dim fuzzy graphs. Each epoch: sample edges (a strong edge fires more often), pull its two endpoints together, and for each pull do a few negative samples — random non-neighbours pushed apart. The learning rate decays to zero, freezing the layout. Springs, but on a sparse graph, so it's fast.
    
        
    for (let epoch=0; epoch<n_epochs; epoch++){
  const alpha = 1.0 * (1 - epoch/n_epochs);       // decaying LR
  for (const e of edges){
    if (Math.random() > e.w) continue;            // sample by weight
    const [i,j] = [e.i, e.j];
    let dx=Y[i][0]-Y[j][0], dy=Y[i][1]-Y[j][1], d=dx*dx+dy*dy;
    const gc = (-2*a*b*Math.pow(d,b-1)) / (1 + a*Math.pow(d,b));   // attract
    Y[i][0]+=clip(gc*dx)*alpha; Y[i][1]+=clip(gc*dy)*alpha;
    Y[j][0]-=clip(gc*dx)*alpha; Y[j][1]-=clip(gc*dy)*alpha;
    for (let s=0;s<5;s++){                         // negative sampling
      const k=(Math.random()*n)|0; if (k===i) continue;
      let rx=Y[i][0]-Y[k][0], ry=Y[i][1]-Y[k][1], rd=rx*rx+ry*ry;
      const rc = (2*b) / ((1e-3+rd)*(1 + a*Math.pow(rd,b)));       // repel
      Y[i][0]+=clip(rc*rx)*alpha; Y[i][1]+=clip(rc*ry)*alpha;
    }
  }
}

    In the real world
    
        
    Nobody hand-rolls UMAP in production. The umap-learn library ships a fast, NN-Descent-backed version; RAPIDS cuML runs it on the GPU. Standardise or PCA down first for speed and denoising, tune n_neighbors (local↔global) and min_dist (tightness), and set a seed for reproducibility. Unlike t-SNE, a fitted UMAP has a .transform() that places new points on the same map — so it doubles as a preprocessing step.
    
        
    # pip install umap-learn
import umap
from sklearn.preprocessing import StandardScaler

Xs = StandardScaler().fit_transform(X)
reducer = umap.UMAP(n_neighbors=15, min_dist=0.1,
                    n_components=2, random_state=42)
emb = reducer.fit_transform(Xs)

import matplotlib.pyplot as plt
plt.scatter(emb[:,0], emb[:,1], c=labels, s=6, cmap="tab10")
