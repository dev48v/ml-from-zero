🌲 Isolation Forest — Fence Off the Odd Points in a Few Random Splits

Anomalies are easy to isolate. Count how many random cuts it takes.

Most points sit inside a dense crowd; a few sit off on their own. Isolation Forest exploits exactly that. Slice the plane with a random cut — pick a feature, pick a split value between its min and max — and repeat, keeping only the side each point falls in, until every point is fenced off alone. A point out in empty space gets isolated in one or two cuts; a point buried in the crowd needs many. Build a whole forest of these random trees, average each point's path length (cuts to isolate it), and turn it into an anomaly score. The heatmap on the left is that score painted over the plane — cool where points are normal, hot where they are anomalous. Drag the threshold to flag the anomalies; the strip on the right shows the punchline: flagged points really do isolate in far fewer splits.

This is a genuine Isolation Forest, run live in your browser. Every tree is built for real: take a small random subsample of ψ points, then recurse — at each node pick one of the two features at random, pick a split value uniformly between that feature's min and max on the points reaching the node, send points left or right, and stop when a point is alone or the tree hits its depth cap of ⌈log₂ψ⌉. A point's path length is the number of splits from the root to where it lands (plus a small correction c(size) for any crowd left unsplit at a capped leaf). Average that over the whole forest to get E(h(x)), normalise by the expected path length c(ψ) of a random binary tree, and the anomaly score is s(x) = 2−E(h(x))/c(ψ). Scores near 1 isolate almost instantly (anomalies); scores near 0.5 sit deep in the crowd (normal). Grow the forest one tree at a time to watch the scores stabilise as the random cuts average out.

vs. density methods (DBSCAN Day 16 / LOF): DBSCAN and LOF hunt anomalies by measuring local density — how many neighbours sit within a radius — which means computing distances between points and choosing a scale (ε, k) that stops working in high dimensions. Isolation Forest never measures density or distance at all: it just counts random cuts, so it runs in near-linear time, needs almost no tuning, thrives in high dimensions, and stays fast on millions of rows. The trade-off: it assumes anomalies are few and different (easy to isolate), so it can miss anomalies hiding in dense local pockets that a density method would catch.

🔨 Build it yourself

    
A random-cut tree, a path-length reader with the c(n) correction, a forest average, and a score. ~60 lines of vanilla JS — then the real-world scikit-learn version in a handful of lines.

    
    
      
  1. 
        
    1
    The setup: unlabelled points, no distances
    
        
    Isolation Forest is unsupervised — you hand it feature vectors and nothing else. There are no class labels and, unlike DBSCAN or k-NN, no distance matrix. Each tree will only ever look at one feature at a time and cut it at a random value, so the algorithm scales to many features and never has to define what "close" means.
    
        
    // X: array of feature vectors, e.g. 2-D points here
const X = points;              // [[x, y], ...]  — no labels, no distances
const N = X.length;
const DIM = X[0].length;       // number of features (2 in this demo)

    2
    Build one isolation tree: random feature, random split
    
        
    The whole tree is random. At each node pick a feature at random, then a split value uniformly between that feature's min and max over the points that reached this node. Send points left or right and recurse. Stop when a point is alone (isolated), when all remaining points are identical, or when you hit a depth cap of ⌈log₂ψ⌉ — deeper than that only matters for normal points, and we do not care exactly how deep they are.
    
        
    function iTree(idx, depth, maxDepth){
  if (depth >= maxDepth || idx.length <= 1)
    return { leaf: true, size: idx.length };
  const q = Math.floor(Math.random() * DIM);          // random feature
  let mn = Infinity, mx = -Infinity;
  for (const i of idx){ const v = X[i][q]; if (vmx) mx=v; }
  if (mn === mx) return { leaf: true, size: idx.length };
  const p = mn + Math.random() * (mx - mn);           // random split value
  const L = [], R = [];
  for (const i of idx) (X[i][q] < p ? L : R).push(i);
  return { leaf:false, q, p, l:iTree(L,depth+1,maxDepth), r:iTree(R,depth+1,maxDepth) };
}

  3. 
        
    3
    Path length: how many cuts to reach the point
    
        
    Drop a point through the tree and count the edges to the leaf it lands in — that is its path length. When a leaf was capped early with more than one point still inside, add c(size): the expected extra depth a full binary tree would have taken to finish isolating that crowd. c(n) uses the harmonic number and is the same formula that later normalises the score.
    
        
    function cFactor(n){                       // avg path length in a random BST
  if (n <= 1) return 0;
  if (n === 2) return 1;
  const H = Math.log(n - 1) + 0.5772156649;      // harmonic ≈ ln + Euler-γ
  return 2 * H - 2 * (n - 1) / n;
}
function pathLength(node, x, depth){
  if (node.leaf) return depth + cFactor(node.size);   // + correction for crowd
  return x[node.q] < node.p
    ? pathLength(node.l, x, depth + 1)
    : pathLength(node.r, x, depth + 1);
}

  4. 
        
    4
    Grow a forest on small subsamples
    
        
    One random tree is noisy, so build many and average. Each tree is trained on a small random subsample of ψ points (256 is the classic default) — subsampling is not just for speed, it actively sharpens anomalies by shrinking the crowd that would otherwise mask them ("swamping"). The depth cap is set from ψ, since a random tree on ψ points reaches typical depth ~log₂ψ.
    
        
    function buildForest(nTrees, psi){
  const maxDepth = Math.ceil(Math.log2(Math.max(2, psi)));
  const trees = [];
  for (let t = 0; t < nTrees; t++){
    const sample = subsample(psi);          // ψ random rows, no replacement
    trees.push(iTree(sample, 0, maxDepth));
  }
  return trees;
}

  5. 
        
    5
    The anomaly score s(x) = 2^(−E(h)/c(ψ))
    
        
    Average a point's path length over every tree to get E(h(x)), then normalise by c(ψ), the expected path length of a random tree on ψ points, so scores are comparable regardless of sample size. The exponential map squashes it to (0, 1): short paths → E(h) small → score near 1 (anomaly); long paths → score near 0.5 (deep in the crowd). Around 0.5 means "no clear anomaly signal."
    
        
    function scoreAll(trees, psi){
  const cN = cFactor(psi);
  return X.map(x => {
    let sum = 0;
    for (const t of trees) sum += pathLength(t, x, 0);
    const eh = sum / trees.length;          // E(h(x)) over the forest
    return Math.pow(2, -eh / cN);            // s(x) in (0, 1)
  });
}

    6
    Threshold / contamination → flag anomalies
    
        
    A score alone is not a decision — you need a cutoff. Set a score threshold directly, or, more commonly, declare a contamination rate (the fraction you expect to be anomalous) and flag the top scores until you hit it. Contamination is just a threshold expressed as a quantile; pick it from domain knowledge or by inspecting the score histogram for a gap.
    
        
    // (a) fixed threshold on the score
const flagged = scores.map(s => s >= 0.60);

// (b) contamination: flag the top c fraction by score
function flagByContamination(scores, c){
  const cut = [...scores].sort((a,b)=>b-a)[Math.floor(c * scores.length)];
  return scores.map(s => s >= cut);
}

    In the real world
    
        
    Nobody hand-rolls this in production. Scikit-learn's IsolationForest does everything above in one fast call: fit builds the forest, predict returns +1 (inlier) / −1 (outlier), and score_samples / decision_function give the raw scores for your own threshold. Set contamination to your expected anomaly rate, leave n_estimators=100 and max_samples=256 as sane defaults, and it scales to millions of rows and hundreds of features.
    
        
    # pip install scikit-learn
from sklearn.ensemble import IsolationForest

clf = IsolationForest(
    n_estimators=100,      # trees in the forest
    max_samples=256,       # ψ — subsample size per tree
    contamination=0.05,    # expected fraction of anomalies
    random_state=0,
)
clf.fit(X)

pred   = clf.predict(X)            # +1 normal, -1 anomaly
scores = -clf.score_samples(X)     # higher = more anomalous
anoms  = X[pred == -1]
