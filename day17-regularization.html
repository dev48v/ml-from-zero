🪢 Regularization — Tame Overfitting With L1 & L2

Watch a penalty on coefficient size turn a wild overfit into a sensible curve

Twelve noisy dots come from a smooth underlying curve. We fit them with a degree-9 polynomial — way too flexible — so with no penalty it wiggles violently through every point: that's overfitting. Regularization adds a term to the loss that punishes large coefficients. Drag λ (lambda) up and the fit smooths toward the true curve; push it too far and it flattens into underfitting. Flip between L2 (Ridge) and L1 (Lasso) and watch the coefficient bars: L2 shrinks them all smoothly, while L1 drives some of them to exactly zero — automatic feature selection.

Top: the data (sky dots), the true curve (dashed gray), and the fitted degree-9 polynomial (sky line). Bottom: the magnitude of each of the 10 coefficients (w₀…w₉) — bars that vanish are coefficients L1 has zeroed out.

🔨 Build it yourself

    
Regularized regression from scratch: build polynomial features, solve Ridge in closed form by adding λI to the normal equations, trace the coefficient path as λ sweeps, implement Lasso with coordinate descent so coefficients hit exactly zero, then pick λ honestly with cross-validation. The same two-line idea you'd reach for in scikit-learn — now you'll know what's underneath it.

    Build polynomial features
    
        
    

    // one input x becomes a row of powers: [1, x, x², …, x^d].
// a high degree d gives the model lots of flexibility — which is
// exactly what lets it overfit, and exactly what we'll rein back in.
function polyFeatures(xs, degree) {
  return xs.map(x => {
    const row = [];
    for (let p = 0; p <= degree; p++) row.push(Math.pow(x, p));
    return row;                       // [x^0, x^1, …, x^degree]
  });
}
const X = polyFeatures(xs, 9);        // 10 columns (w0…w9)

    Standardize the features first
    
        
    

    # x⁹ is astronomically larger than x¹, so a single λ would punish
# them unfairly. center & scale each column to mean 0, std 1 so the
# penalty hits every coefficient on equal footing. (don't penalise
# the intercept — fit it separately or on centered targets.)
mu, sd = X.mean(0), X.std(0)
Xs = (X - mu) / sd            # every column now ~ N(0, 1)
# always standardize before Ridge/Lasso — it's not optional.

  3. 
        
    3
    Ridge (L2) in closed form: add λI
    
        
    

    // ordinary least squares solves  w = (XᵀX)⁻¹ Xᵀy.
// Ridge adds λ·‖w‖² to the loss, and the solution barely changes:
//        w = (XᵀX + λI)⁻¹ Xᵀy
// that +λI bumps the diagonal, shrinking every coefficient toward 0
// AND it makes the matrix invertible even when XᵀX is singular.
function ridge(X, y, lambda) {
  const XtX = matMul(transpose(X), X);
  for (let i = 0; i < XtX.length; i++) XtX[i][i] += lambda;   // + λI
  const Xty = matVec(transpose(X), y);
  return solve(XtX, Xty);             // Gaussian elimination
}

# the "regularization path" shows coefficients shrink toward 0 as λ

import numpy as np
lambdas = np.logspace(-4, 2, 50)      # 1e-4 … 1e2
path = np.array([ridge(Xs, y, lam) for lam in lambdas])

    Lasso (L1) via coordinate descent
    
        
    

    // L1 has no clean closed form — the |w| kink isn't differentiable.
// coordinate descent fixes that: optimise ONE coefficient at a time,
// each via a soft-threshold that can snap it to EXACTLY zero.
function softThreshold(rho, lam) {        // the sparsity-maker
  if (rho >  lam) return rho - lam;
  if (rho < -lam) return rho + lam;
  return 0;                               // |rho| <= lam  →  exactly 0
}
function lasso(X, y, lambda, iters = 200) {
  const n = X.length, d = X[0].length, w = new Array(d).fill(0);
  for (let t = 0; t < iters; t++)
    for (let j = 0; j < d; j++) {
      // residual ignoring feature j, then its correlation rho
      let rho = 0;
      for (let i = 0; i < n; i++) {
        let pred = 0;
        for (let k = 0; k < d; k++) if (k !== j) pred += X[i][k]*w[k];
        rho += X[i][j] * (y[i] - pred);
      }
      w[j] = softThreshold(rho / n, lambda);   // may become 0
    }
  return w;                                // many entries land on 0
}

    Pick λ by cross-validation
    
        
    

    # λ is a knob YOU choose — and the honest way is k-fold CV.
# for each candidate λ, train on k-1 folds, score the held-out fold,
# average. the λ with the lowest mean validation error wins.
from sklearn.model_selection import cross_val_score
best_lam, best_score = None, -1e9
for lam in lambdas:
    score = cross_val_score(Ridge(alpha=lam), Xs, y,
                            cv=5, scoring="neg_mean_squared_error").mean()
    if score > best_score: best_score, best_lam = score, lam
# never pick λ on the training error — it always prefers λ = 0.

    In the real world: scikit-learn
    
        
    

    from sklearn.linear_model import Ridge, Lasso, ElasticNet
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline

# Ridge: shrinks everything, keeps all features (alpha == λ)
ridge = make_pipeline(StandardScaler(), Ridge(alpha=1.0)).fit(X, y)

# Lasso: zeroes weak features → built-in feature selection
lasso = make_pipeline(StandardScaler(), Lasso(alpha=0.1)).fit(X, y)
print((lasso[-1].coef_ != 0).sum(), "features kept")

# ElasticNet: blend of both via l1_ratio (1=Lasso, 0=Ridge)
enet = ElasticNet(alpha=0.1, l1_ratio=0.5).fit(X, y)
