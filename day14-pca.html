📉 PCA — Compress Data, Keep the Signal

Watch PCA find the directions the data actually spreads

🔨 Build it yourself

    
PCA from scratch: centre the data, build the covariance matrix, take its eigenvectors (the principal directions), sort them by eigenvalue, and project onto the top ones. Five honest steps and you've reduced the dimensions.

    
    
      
  1. 
        
    1
    The data: rows of features (no labels needed)
    
        
    

    // each row is one sample, each column is one feature.
// PCA is unsupervised — it only looks at how the FEATURES
// move, never at any target/answer column.
const X = [
  [2.5, 2.4], [0.5, 0.7], [2.2, 2.9],
  [1.9, 2.2], [3.1, 3.0], [2.3, 2.7],
  // ... n samples × d features
];
    
          
        
    
      
    2. 
      
  2. 
        
    2
    Centre the data (subtract each feature's mean)
    
        
    

    // PCA measures spread AROUND the centre, so shift the cloud
// so its mean sits at the origin. (Scaling each feature to
// unit variance first is also standard — do it when units differ.)
const n = X.length, d = X[0].length;
const mean = Array(d).fill(0);
X.forEach(row => row.forEach((v, j) => mean[j] += v / n));
const Xc = X.map(row => row.map((v, j) => v - mean[j]));
    
          
        
    
      
    3. 
      
  3. 
        
    3
    Build the covariance matrix
    
        
    

    // cov[i][j] = how feature i and feature j vary TOGETHER.
// the diagonal is each feature's own variance; off-diagonals
// are positive if they rise together, negative if they trade off.
function covariance(Xc) {
  const n = Xc.length, d = Xc[0].length;
  const C = Array.from({length: d}, () => Array(d).fill(0));
  for (const row of Xc)
    for (let i = 0; i < d; i++)
      for (let j = 0; j < d; j++)
        C[i][j] += row[i] * row[j] / (n - 1);   // n-1 = sample cov
  return C;
}
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Eigen-decompose: closed form for a 2×2
    
        
    

    // eigenVECTORS = the principal directions; eigenVALUES = the
// variance along each. for a symmetric 2×2 [[a,b],[b,c]] there's
// a tidy closed form (real libraries use SVD for any size).
function eig2x2(a, b, c) {
  const tr = a + c, det = a * c - b * b;
  const disc = Math.sqrt(Math.max(0, tr*tr/4 - det));
  const l1 = tr/2 + disc, l2 = tr/2 - disc;     // l1 ≥ l2
  // eigenvector for l1 (handle b≈0 → axis-aligned)
  let v1 = Math.abs(b) > 1e-12 ? [l1 - c, b] : (a >= c ? [1,0] : [0,1]);
  const m = Math.hypot(v1[0], v1[1]); v1 = [v1[0]/m, v1[1]/m];
  const v2 = [-v1[1], v1[0]];                   // ⟂ to v1
  return { values: [l1, l2], vectors: [v1, v2] };
}

    5
    Sort by eigenvalue + variance explained
    
        
    

    // biggest eigenvalue = most informative direction → keep it first.
const { values, vectors } = eig2x2(C[0][0], C[0][1], C[1][1]);
const order = [0, 1].sort((i, j) => values[j] - values[i]);
const total = values[0] + values[1];

// fraction of total variance each PC carries (sums to ~1.0)
const explained = order.map(i => values[i] / total);
// keep the top-k whose explained variance covers, say, 95%.

  6. 
        
    6
    Project onto the top components
    
        
    

    // to reduce d→k, dot each centred row with the top-k eigenvectors.
// the result is the same point expressed in the new PC coordinates.
const dot = (a, b) => a.reduce((s, v, i) => s + v * b[i], 0);
const topVecs = order.slice(0, k).map(i => vectors[i]);

const reduced = Xc.map(row => topVecs.map(v => dot(row, v)));
// reduced[i] is now k numbers instead of d — the compression.

// reconstruct (lossy): rebuild in original space, then un-centre.
const recon = reduced.map(z =>
  mean.map((mj, j) => mj + topVecs.reduce((s, v, t) => s + z[t]*v[j], 0)));
// recon ≈ X. what's missing is the variance of the dropped PCs.

    In the real world: scikit-learn
    
        
    

    from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

X = StandardScaler().fit_transform(X)    # SCALE first — units matter!

pca = PCA(n_components=2)                 # or 0.95 to keep 95% variance
Z = pca.fit_transform(X)                  # the reduced data

pca.components_                # the principal directions (eigenvectors)
pca.explained_variance_ratio_ # fraction of variance per PC (≈ sums to 1)
X_back = pca.inverse_transform(Z)         # lossy reconstruction
