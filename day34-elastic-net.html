🕸️ Elastic Net — Blend L1 + L2 to Get Sparse and Stable Coefficients

Two of these features are near-duplicates (correlation 0.98) and four are pure noise. Drag the two sliders and watch coordinate descent recompute every coefficient live — see lasso arbitrarily kill one of the correlated twins, ridge refuse to zero anything, and Elastic Net do the thing both miss.

A plain least-squares fit gives every one of the eight features a weight — including the four that are just noise, and it splits the two correlated twins (x₁, x₂) randomly. Regularization fixes that by adding a penalty on the coefficients. Ridge (L2) shrinks them all smoothly and shares weight fairly between correlated features, but never drives one to exactly zero. Lasso (L1) zeros features for a sparse, self-selecting model — but faced with two correlated twins it keeps one at random and discards the other, and that pick flips with tiny data changes. Elastic Net adds both penalties: alpha sets the total strength, l1_ratio mixes them (0 = pure ridge … 1 = pure lasso). The L1 part zeros the noise (sparsity) while the L2 part keeps the correlated pair together (the grouping effect). Every coefficient below is genuinely solved by coordinate descent + soft-thresholding running live in your browser — no libraries.

🔨 Build it yourself

    
scikit-learn ships Elastic Net as ElasticNet (and ElasticNetCV to pick the knobs for you). Its alpha is the total penalty strength and l1_ratio is the mix — exactly the two sliders in the demo. Below is the whole workflow: standardize, fit, pick alpha/l1_ratio by cross-validation, read the coefficients, and the from-scratch coordinate descent that produced every number you just dragged.

    Standardize first — penalties are scale-sensitive
    
        
    L1 and L2 penalize the size of each coefficient, so a feature measured in millimetres gets a bigger coefficient (and a harsher penalty) than the same feature in metres. Always put the features on a common scale first, inside a Pipeline so the scaler is refit per CV fold and never leaks.
    
        
    from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import ElasticNet
from sklearn.pipeline import make_pipeline

# zero mean, unit variance -> every coefficient is penalized on equal footing
model = make_pipeline(StandardScaler(), ElasticNet(alpha=0.4, l1_ratio=0.5))
model.fit(X_train, y_train)

    Understand the two knobs: alpha and l1_ratio
    
        
    The Elastic Net objective is the least-squares loss plus a blended penalty. alpha scales the whole penalty (0 = ordinary least squares); l1_ratio splits it between L1 and L2. Set l1_ratio=1 and you have Lasso; set l1_ratio=0 and you have Ridge.
    
        
    # minimize over w:
#   (1 / 2n) * ||y - Xw||^2                      <- least-squares fit
#   + alpha * l1_ratio       * ||w||_1           <- L1: sparsity
#   + 0.5 * alpha * (1 - l1_ratio) * ||w||_2^2   <- L2: shrink + grouping

ElasticNet(alpha=0.0)               # == LinearRegression (no penalty)
ElasticNet(alpha=a, l1_ratio=1.0)   # == Lasso(alpha=a)
ElasticNet(alpha=a, l1_ratio=0.0)   # == Ridge  (2*a on the L2 term)

    Let cross-validation pick alpha AND l1_ratio
    
        
    Don't guess the knobs — sweep them. ElasticNetCV tries a grid of l1_ratio values and, for each, an automatically-chosen path of alphas, scoring every combination with k-fold CV and keeping the best.
    
        
    from sklearn.linear_model import ElasticNetCV

cv = ElasticNetCV(
    l1_ratio = [.1, .3, .5, .7, .9, .95, 1.0],  # grid of mixes to try
    alphas   = None,      # None -> sklearn builds a sensible alpha path
    cv       = 5,
    n_jobs   = -1,
).fit(X_train_scaled, y_train)

print("best alpha    :", cv.alpha_)
print("best l1_ratio :", cv.l1_ratio_)

    Coordinate descent from scratch — the exact demo engine
    
        
    Elastic Net has no closed form because of the L1 kink, so the standard solver cycles one coefficient at a time. For each feature it computes a partial residual correlation rho, applies soft-thresholding (that is what creates the exact zeros), then divides by the L2 term. This is the whole algorithm sklearn uses — and the one running in this page.
    
        
    import numpy as np

def soft_threshold(x, g):          # the L1 operator -> makes exact zeros
    return np.sign(x) * max(abs(x) - g, 0.0)

def elastic_net(X, y, alpha, l1_ratio, iters=500):
    X = (X - X.mean(0)) / X.std(0)          # standardize
    y = y - y.mean()                        # center (handle intercept apart)
    n, p = X.shape
    w = np.zeros(p)
    r = y - X @ w                           # residual
    thr   = alpha * l1_ratio                # L1 shrink amount
    denom = 1.0 + alpha * (1 - l1_ratio)    # L2 divisor  (col var = 1)
    for _ in range(iters):
        for j in range(p):
            rho = (X[:, j] @ (r + X[:, j] * w[j])) / n   # add j back in
            new = soft_threshold(rho, thr) / denom
            r += X[:, j] * (w[j] - new)     # keep residual in sync
            w[j] = new
    return w                                # zeros = dropped features

    Choose the right tool: ridge vs lasso vs elastic net
    
        
    Elastic Net is not always the answer. Reach for it specifically when you have many, correlated features and want sparsity you can trust. If nothing is correlated, plain Lasso is simpler; if you want to keep every feature and only tame their size, Ridge is enough.
    
        
    # rules of thumb
# Ridge  (l1_ratio=0)  : keep ALL features, just shrink; great for
#                        multicollinearity when you don't need selection.
# Lasso  (l1_ratio=1)  : want a sparse model; features roughly independent.
#                        Caveat: with p > n it selects at most n features,
#                        and among correlated features it picks ONE at random.

🎉 Day 34 of MachineLearningFromZero done.
You watched coordinate descent recompute eight coefficients live as you mixed two penalties. L2 (ridge) shrinks everything and shares weight between correlated features but never selects; L1 (lasso) zeros features for a sparse model but picks arbitrarily among correlated twins and caps out at n features. Elastic Net blends them: the L1 part kills the noise features while the L2 part keeps the correlated pair together — the grouping effect — which is why it beat pure lasso on the held-out test set here. You learned the objective (alpha for strength, l1_ratio for the mix), why standardizing first matters, that soft-thresholding is what manufactures the exact zeros, and to let ElasticNetCV choose the knobs. 👉 Tomorrow — Day 35: Bayesian Linear Regression — put a distribution over the weights and get calibrated uncertainty on every prediction, not just a point estimate.
