MachineLearningFromZero · Day 43

🐱 CatBoost — Beating Target Leakage with Order: Ordered Target Statistics Encode Categoricals Using Only Earlier Rows in a Random Permutation, Ordered Boosting Computes Each Row's Residual From a Model That Never Saw It, and Symmetric (Oblivious) Trees Keep Inference Fast

Day 40 gave gradient boosting a second-order objective (XGBoost); Day 41 made the same boosting fast (LightGBM). CatBoost completes the trio by attacking a subtler enemy: target leakage. Two places leak the label into training — encoding a category by the mean of its target (a row peeks at its own answer), and computing a boosting residual from a model that was trained on that same row. Both inflate training scores and quietly overfit. CatBoost's fix is ordering: process rows in a random permutation and, for any row, use only the rows that came before it. The result is honest, leak-free training signal — plus symmetric (oblivious) trees that split on the same feature/threshold across a whole level, giving 2ᵈ balanced leaves, branch-free inference, and built-in regularization. Everything below is computed live in JS, no library.

Pick a demo. A normal (CART) tree picks the best split at every node independently, so nodes at the same depth split on different rules — flexible but easy to overfit. A symmetric / oblivious tree (CatBoost's only tree type) picks one (feature, threshold) per level and applies it to all nodes there. A depth-d oblivious tree is exactly 2ᵈ balanced leaves indexed by d yes/no tests — inference is d comparisons with no branching, and the rigid shape is a strong regularizer.
🌿 NORMAL TREE — per-node splits
Each node free to choose its own split → jagged, asymmetric regions.
🧊 OBLIVIOUS TREE — one split per level
Same rule across each level → a clean grid of 2ᵈ cells. This is what CatBoost grows.

    Naive target statistics — and why it leaks
    
        
    Replace a category by the mean target over all its rows. Because the sum includes the current row, a category that appears once encodes to (essentially) its own label. That is target leakage: the feature is now a disguised copy of y.
    
        
    import numpy as np

def naive_target_stats(cat, y, a=1.0, prior=None):
    """Encode each row by the mean target of its category — INCLUDING itself."""
    prior = y.mean() if prior is None else prior
    sums, cnts = {}, {}
    for c, t in zip(cat, y):                 # totals include every row
        sums[c] = sums.get(c, 0.0) + t
        cnts[c] = cnts.get(c, 0) + 1
    enc = np.array([(sums[c] + a*prior) / (cnts[c] + a) for c in cat])
    return enc

# a category that appears once -> enc = (y + a*prior)/(1+a)  -> threshold at 0.5 nails y
# corr(enc, y) climbs toward 1.0 as cardinality rises: the feature IS the label

    Ordered target statistics — use only earlier rows
    
        
    Fix a random permutation. Encode each row with the running mean of the target over earlier same-category rows only, smoothed by a prior. No row ever contributes to its own encoding, so the leak is gone — yet the genuine category signal survives.
    
        
    def ordered_target_stats(cat, y, a=1.0, prior=None, seed=0):
    """Encode row i with earlier same-category rows in a random permutation."""
    prior = y.mean() if prior is None else prior
    n = len(cat)
    perm = np.random.default_rng(seed).permutation(n)
    enc = np.zeros(n)
    seen_sum, seen_cnt = {}, {}
    for i in perm:                           # walk rows in permutation order
        c = cat[i]
        s, k = seen_sum.get(c, 0.0), seen_cnt.get(c, 0)
        enc[i] = (s + a*prior) / (k + a)     # PRIOR rows only -> no self-leak
        seen_sum[c] = s + y[i]               # only now add this row
        seen_cnt[c] = k + 1
    return enc
# CatBoost averages several permutations to cut the variance of early rows.
# At inference time it uses the full-data mean (targets are unknown -> no leak).

    Measure the leak — train vs validation
    
        
    Train a tiny model on each encoding and compare. The naive encoding scores near-perfectly on train but collapses to the base rate on validation (a huge gap = overfitting). Ordered TS keeps train ≈ val — it generalizes.
    
        
    from sklearn.linear_model import LogisticRegression

def gap(enc_train, enc_val, y_tr, y_va):
    m = LogisticRegression().fit(enc_train.reshape(-1,1), y_tr)
    tr = m.score(enc_train.reshape(-1,1), y_tr)
    va = m.score(enc_val.reshape(-1,1),   y_va)
    return tr, va, tr - va

# naive :  train 0.98  val 0.55   gap 0.43   <- leaked; the score is a mirage
# ordered: train 0.61  val 0.58   gap 0.03   <- honest; survives to validation
# The gap, not the raw train score, is the signal you must watch.

    Ordered boosting — a residual from a model that never saw the row
    
        
    The same leak appears in boosting: a residual y−F(x) is biased because F trained on that row (prediction shift). Ordered boosting builds each row's gradient from a model fit only on rows earlier in the permutation. The out-of-fold version below captures the identical leak-free principle.
    
        
    def leak_free_residuals(x, y, F, folds):
    """Each row's residual comes from a stump NOT trained on its fold."""
    upd = np.zeros_like(F)
    for k in np.unique(folds):
        tr = folds != k                      # everything except fold k
        stump = fit_stump(x[tr], (y - F)[tr])# fit on the OTHER rows
        upd[folds == k] = stump(x[folds == k])  # predict the held-out fold
    return upd                               # F never influenced its own target

# greedy boosting: residual = y - F(x)      with F trained INCLUDING x  -> shift
# ordered boosting: residual from a prefix/out-of-fold model -> unbiased gradient
# CatBoost keeps ~log(n) prefix models per permutation so the cost stays sane.

    The oblivious (symmetric) tree — one split per level, branch-free inference
    
        
    CatBoost grows only symmetric trees: every node at a level uses the same (feature, threshold). A depth-d tree is 2ᵈ leaves indexed by d bit-tests, so scoring is d comparisons with no branch mispredictions — and the rigid shape resists overfitting.
    
        
    def fit_oblivious(X, y, depth):
    levels = []
    for _ in range(depth):
        # pick ONE (feature, threshold) that best splits ALL current groups
        best = min(candidate_splits(X),
                   key=lambda ft: total_gini_after(X, y, levels, ft))
        levels.append(best)                  # same rule for the whole level
    leaves = {}                              # 2**depth leaves keyed by bit-index
    for xi, yi in zip(X, y):
        leaves.setdefault(leaf_index(xi, levels), []).append(yi)
    return levels, {k: np.mean(v) for k, v in leaves.items()}

def leaf_index(x, levels):
    idx = 0
    for f, thr in levels:                    # d comparisons, no branching
        idx = (idx << 1) | int(x[f] >= thr)
    return idx

🎉 Day 43 of MachineLearningFromZero done — the GBDT trio is complete.
CatBoost keeps the exact g, h gradient-boosting engine of XGBoost and LightGBM but obsesses over target leakage. Ordered target statistics encode a category with the running target mean of only the rows that came before it in a random permutation, so no row ever peeks at its own label — killing the leak that naive mean-encoding creates on high-cardinality features. Ordered boosting applies the same trick to gradients: each row's residual is produced by a model that never trained on it, removing the "prediction shift" that biases ordinary boosting. And symmetric (oblivious) trees split on one (feature, threshold) per level, giving 2ᵈ balanced leaves, branch-free d-comparison inference, and a built-in regularizer. The payoff: state-of-the-art results on messy categorical data with almost no tuning — pass raw columns via cat_features and trust the defaults. 👉 Together with XGBoost (Day 40) and LightGBM (Day 41), CatBoost rounds out the three boosters that win most tabular competitions.
