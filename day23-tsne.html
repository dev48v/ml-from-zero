🗺️ t-SNE — Squash High-Dim Data to 2D

Watch six-dimensional blobs untangle themselves into a 2D map.

The points below live in 6 dimensions — four Gaussian blobs you can't see because your screen is flat. t-SNE finds a 2D position for every point so that neighbours in 6D stay neighbours on the map. It measures how similar every pair of points is in high-dim (a probability, controlled by perplexity), then nudges the 2D points by gradient descent until their similarities match — minimising the KL divergence. The dots start as one random blob at the centre; press Run and watch the four true clusters (their real colours) pull apart as the KL number falls.

This is a genuine, if small, t-SNE: per-point Gaussian bandwidths are binary-searched to hit the target perplexity, the joint high-dim affinities P are symmetrised, the low-dim affinities Q use the Student-t kernel 1/(1+d²), and the 2D positions descend the true KL(P‖Q) gradient with momentum. Colours mark the true cluster so you can judge the result — but t-SNE never sees them. KL should fall from ~1.6 to ~0.1 and stay down. Try dragging perplexity: small values chase tiny local neighbourhoods; large values smear clusters together.

🔨 Build it yourself

    
Pairwise distances, a perplexity-tuned Gaussian for the high-dim affinities P, a Student-t kernel for the low-dim Q, the KL gradient, and a momentum descent loop. ~90 lines of vanilla JS — then the two-line real-world version.

    1
    High-dim affinities with perplexity
    
        
    For each point, turn distances to every other point into probabilities with a Gaussian. The bandwidth (β = 1/2σ²) is binary-searched so the row's entropy matches log(perplexity) — that pins the effective neighbourhood size. Dense regions get a narrow Gaussian, sparse regions a wide one.
    
        
    function pRow(D, i, perplexity){          // D[i][j] = squared dist
  let beta=1, lo=-Infinity, hi=Infinity, P=[];
  const logU = Math.log(perplexity);
  for (let t=0;t<50;t++){
    let sum=0; P = D[i].map((d,j)=> i===j?0:Math.exp(-d*beta));
    P.forEach(v=>sum+=v); sum=sum||1e-12;
    let H=0; P.forEach(v=>{ const p=v/sum; if(p>1e-12) H-=p*Math.log(p); });
    const diff = H - logU;
    if (Math.abs(diff)<1e-4) break;
    if (diff>0){ lo=beta; beta = hi===Infinity?beta*2:(beta+hi)/2; }
    else       { hi=beta; beta = lo===-Infinity?beta/2:(beta+lo)/2; }
  }
  const s = P.reduce((a,b)=>a+b,0)||1e-12;
  return P.map(v=>v/s);
}

    Symmetrise into the joint P
    
        
    The per-row probabilities are asymmetric (i's view of j ≠ j's view of i). t-SNE symmetrises them into a single joint distribution P over pairs, then normalises so every entry sums to 1. A tiny floor keeps values away from zero so the log in KL never blows up.
    
        
    function joint(X, perplexity){
  const n = X.length, D = dists(X);
  const rows = X.map((_,i)=>pRow(D,i,perplexity));
  const P = Array.from({length:n},()=>new Float64Array(n));
  let tot=0;
  for (let i=0;i<n;i++) for (let j=0;j<n;j++){
    P[i][j] = rows[i][j] + rows[j][i]; tot += P[i][j];
  }
  for (let i=0;i<n;i++) for (let j=0;j<n;j++)
    P[i][j] = Math.max(P[i][j]/tot, 1e-12);   // normalise + floor
  return P;
}

    Low-dim affinities Q (Student-t)
    
        
    In the 2D map, similarity uses a Student-t kernel with one degree of freedom: 1/(1+d²). Its heavy tail lets far-apart points stay far apart cheaply, which cures the "crowding problem" — the reason a plain Gaussian map jams everything into the middle. Keep the un-normalised numerators around; the gradient reuses them.
    
        
    function qLowDim(Y){
  const n=Y.length, num=Array.from({length:n},()=>new Float64Array(n));
  let sum=0;
  for (let i=0;i<n;i++) for (let j=0;j<n;j++){
    if (i===j) continue;
    const v = 1/(1 + d2(Y[i],Y[j]));   // Student-t, heavy tail
    num[i][j]=v; sum+=v;
  }
  const Q=Array.from({length:n},()=>new Float64Array(n));
  for (let i=0;i<n;i++) for (let j=0;j<n;j++)
    Q[i][j] = Math.max(num[i][j]/(sum||1e-12), 1e-12);
  return { Q, num };
}

    The KL gradient
    
        
    The cost is KL(P‖Q) — how badly the map's similarities disagree with the true ones. Its gradient has a clean form: each point is pulled toward pairs where P>Q (should be closer) and pushed from pairs where Q>P (should be farther), scaled by the Student-t numerator. Springs, essentially — attract true neighbours, repel the rest.
    
        
    function grad(P,Q,num,Y){
  const n=Y.length, g=Y.map(()=>[0,0]);
  for (let i=0;i<n;i++) for (let j=0;j<n;j++){
    if (i===j) continue;
    const m = (P[i][j]-Q[i][j])*num[i][j];   // (p−q)/(1+d²)
    g[i][0] += 4*m*(Y[i][0]-Y[j][0]);
    g[i][1] += 4*m*(Y[i][1]-Y[j][1]);
  }
  return g;
}

    Descend with momentum
    
        
    Start the map as a tiny random blob near the origin, then step downhill. Momentum (blend last step into this one) smooths the ride and escapes shallow traps. Re-centre each iteration so the whole cloud doesn't drift. Watch KL fall and the clusters snap apart.
    
        
    let Y = X.map(()=>[randn()*1e-2, randn()*1e-2]);
let vel = Y.map(()=>[0,0]);
for (let it=0; it<500; it++){
  const { Q, num } = qLowDim(Y);
  const g = grad(P, Q, num, Y);
  const mom = it<100 ? 0.5 : 0.7;
  for (let i=0;i<Y.length;i++){
    vel[i][0] = mom*vel[i][0] - 120*g[i][0];
    vel[i][1] = mom*vel[i][1] - 120*g[i][1];
    Y[i][0]+=vel[i][0]; Y[i][1]+=vel[i][1];
  }
  recentre(Y);
}

    In the real world
    
        
    Nobody hand-rolls t-SNE in production. scikit-learn ships an optimised, Barnes-Hut version; openTSNE is faster and supports adding new points. Always standardise or PCA down to ~30–50 dims first (t-SNE is O(N²) and slow on raw high-dim data), then run it. Tune perplexity in the 5–50 range and set a seed for reproducibility.
    
        
    # pip install scikit-learn        (or:  pip install openTSNE)
from sklearn.manifold import TSNE
from sklearn.decomposition import PCA

X50 = PCA(n_components=50).fit_transform(X)      # speed + denoise
emb = TSNE(n_components=2, perplexity=30,
           init="pca", random_state=42).fit_transform(X50)

import matplotlib.pyplot as plt
plt.scatter(emb[:,0], emb[:,1], c=labels, s=6, cmap="tab10")
