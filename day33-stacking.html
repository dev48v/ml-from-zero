🏗️ Stacking — Let a Meta-Model Learn How to Combine Diverse Base Models

A linear model, a k-NN, and a shallow tree each get this boundary partly wrong — but they get different parts wrong. Stacking trains a meta-model to learn who to trust where, and it beats every single one of them.

Three diverse base learners are fit on the same two-moons data below — a logistic regression (only draws a line), a k-NN (local, noisy near the seam), and a depth-2 tree (only axis-aligned boxes). Because their mistakes differ, a fourth model can do better than any of them — but only if you feed it the bases' predictions the right way. The trick is out-of-fold prediction: split the training rows into k folds, and for each fold let the bases predict it after training on the other folds. Every row's meta-features then come from a base that never saw that row — no leakage. A logistic-regression meta-model learns weights over those held-out predictions and produces the blended boundary. Everything here — the three base fits, the k-fold cross-val predictions, and the meta fit — runs live in your browser, no libraries.

THE OUT-OF-FOLD TRICK — how the meta-features are built without leakage
training rows coloured by fold

Split the training set into 5 folds. Hold one fold out; fit every base on the rest; let them predict the held-out fold. Rotate through all folds → every training row gets base predictions from models that never trained on it. Those held-out probabilities become the meta-model's features.

⚠️ Why not just predict the training rows directly? A base scoring its own training data is a mirage. The leaky meta below looks like — on the data it was fit on, but held out it is only —. Out-of-fold predictions report the honest number before you deploy.
TEST ACCURACY — bases vs the stack
HOW MUCH THE META-MODEL TRUSTS EACH BASE (learned weights)
The meta-model is free to learn any weighting — including a negative weight, meaning it uses a weak base as a corrector rather than trusting it directly. That flexibility is what separates stacking from a plain average vote.

🔨 Build it yourself

    
scikit-learn ships stacking as StackingClassifier (and StackingRegressor). It does the out-of-fold plumbing for you: its cv argument is exactly the k you set in the demo, and the held-out predictions become the final_estimator's training features. Below is the whole workflow — diverse bases, a simple meta, honest evaluation.

    
    
      
  1. 
        
    1
    Pick DIVERSE base estimators
    
        
    Diversity is the fuel. Choose models that make different mistakes — a linear model, a distance model, a tree/forest, a boosting model. Near-identical learners give the meta-model nothing to arbitrate.
    
        
    from sklearn.linear_model  import LogisticRegression
from sklearn.neighbors     import KNeighborsClassifier
from sklearn.tree          import DecisionTreeClassifier
from sklearn.ensemble      import RandomForestClassifier

estimators = [
    ("lr",   LogisticRegression(max_iter=1000)),  # linear, global
    ("knn",  KNeighborsClassifier(n_neighbors=7)), # local
    ("tree", DecisionTreeClassifier(max_depth=3)), # axis-aligned
    ("rf",   RandomForestClassifier(n_estimators=200)),
]
    
      
    2. 
      
  2. 
        
    2
    Wire up StackingClassifier — cv IS the out-of-fold
    
        
    The final_estimator is your level-1 meta-model — keep it simple (a logistic regression). cv=5 tells sklearn to build the meta features by 5-fold out-of-fold prediction, so there is no leakage and you do not have to code the fold loop yourself.
    
        
    from sklearn.ensemble import StackingClassifier
from sklearn.linear_model import LogisticRegression

stack = StackingClassifier(
    estimators      = estimators,
    final_estimator = LogisticRegression(),   # the meta / level-1 model
    cv              = 5,                       # <- out-of-fold folds
    stack_method    = "predict_proba",        # feed probabilities, not hard labels
    passthrough     = False,                   # True = also give meta the raw X
    n_jobs          = -1,
)
    
      
    3. 
      
  3. 
        
    3
    Fit, and compare to the bases AND a plain vote
    
        
    Stacking only earns its complexity if it beats (a) every base alone and (b) a simple VotingClassifier average. Always run that comparison on the same untouched test set.
    
        
    from sklearn.model_selection import train_test_split
from sklearn.ensemble import VotingClassifier

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.3,
                                          stratify=y, random_state=0)
stack.fit(X_tr, y_tr)
vote = VotingClassifier(estimators, voting="soft").fit(X_tr, y_tr)

print("stack :", stack.score(X_te, y_te))
print("vote  :", vote.score(X_te, y_te))
for name, est in stack.named_estimators_.items():
    print(name, ":", est.score(X_te, y_te))   # each base alone
    
      
    4. 
      
  4. 
        
    4
    Inspect what the meta-model learned
    
        
    The fitted final_estimator_ holds the weights over the base outputs — read them to see which base the meta trusts, and which it down-weights or uses as a corrector (a negative coefficient, exactly like the demo).
    
        
    meta = stack.final_estimator_
print("meta coefficients:", meta.coef_)     # weight per base (per class)
print("meta intercept   :", meta.intercept_)
# large + weight  -> trusted base
# near-zero / negative -> down-weighted or used to correct another base
    
      
    5. 
      
  5. 
        
    5
    Cross-validate the whole stack honestly
    
        
    The stack has an internal cv for its meta features; wrap the entire stack in an outer cross-validation to estimate real performance. Nesting like this keeps the outer test folds untouched by any part of training.
    
        
    from sklearn.model_selection import cross_val_score, StratifiedKFold

outer = StratifiedKFold(n_splits=5, shuffle=True, random_state=0)
scores = cross_val_score(stack, X, y, cv=outer, scoring="accuracy")
print(scores.mean(), "+/-", scores.std())
# inner cv builds meta features; outer cv scores the finished stack
    
      
    6. 
      
  6. 
        
    6
    Know when NOT to stack
    
        
    Stacking multiplies training cost (every base is refit k+1 times) and adds a model to maintain. If a single tuned gradient-booster already wins, or your bases are correlated, skip it. Reach for it when you have several genuinely different strong models and the last fraction of a percent matters.
    
        
    # quick sanity checks before committing to a stack
# 1) are the bases actually diverse?
import numpy as np
P = np.column_stack([e.predict(X_te) for _, e in stack.named_estimators_.items()])
print("base prediction correlation:\n", np.corrcoef(P.T))  # want LOW off-diagonal

# 2) does stacking beat the best base by more than the noise?
#    if not, ship the single model — it is cheaper and simpler.

🎉 Day 33 of MachineLearningFromZero done.
You saw three diverse base learners make different mistakes, and a logistic-regression meta-model learn to combine them into a boundary that beat all three. You learned the out-of-fold trick — k-fold predictions so every meta-feature comes from a base that never saw that row — and why skipping it (in-sample predictions) fakes a great score that collapses on held-out data. Bagging cuts variance, boosting cuts bias, voting averages blindly; stacking learns the combination, which is why it tops so many Kaggle leaderboards — at the cost of extra compute and a model to maintain. 👉 Tomorrow — Day 34: Elastic-Net — blend L1 and L2 penalties to get sparse, stable coefficients when features are many and correlated.
