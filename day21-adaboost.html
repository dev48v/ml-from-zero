🚀 AdaBoost — Turning Weak Stumps Into a Strong Classifier

Add rounds and watch a pile of dumb one-line rules harden into a real boundary

Left: a 2-class scatter — rose is class −1, sky is class +1 — that no single straight cut can separate. AdaBoost's only tool is a decision stump: a depth-1 rule that splits on one axis at one threshold (an alone-standing horizontal or vertical line). One stump is barely better than a coin flip. The trick is to add many, each one focused on the mistakes of the last. Every point carries a sample weight (drawn as its size); we start them equal, then each round we pick the stump with the lowest weighted error, score it with α = ½·ln((1−err)/err), and multiply up the weight of every point it got wrong so the next stump is forced to look at the hard cases. The final classifier is the weighted vote sign(Σ α·stump). Drag the rounds slider and watch the shaded decision region — built live from the real weighted-error / α loop — go from a single crude slab to a sharp, blocky boundary while train accuracy climbs. Nothing here is faked: every error, α and weight update is computed from your data.

The shaded background is the combined region the strong classifier calls +1 vs −1. The bright white line is the latest round's stump; faint white lines are earlier stumps. Point size = its current sample weight — misclassified points swell so the next stump chases them. A ✕ ring marks points the current ensemble still gets wrong.

Write AdaBoost from scratch with nothing but decision stumps: start with uniform sample weights, then repeat T times — find the stump with the lowest weighted error, turn that error into a voting weight α = ½·ln((1−err)/err), multiply up the weight of every misclassified point and renormalise so the weights stay a distribution.     A weak learner: the decision stump
    
        
    

    # a STUMP is a depth-1 tree: split on ONE feature at ONE threshold.
# it predicts +1 on one side, -1 on the other (a "polarity" flips which).
# on its own it's barely better than a coin flip -> a "weak learner".
import numpy as np

def stump_predict(X, dim, thresh, polarity):
    pred = np.ones(len(X))
    if polarity == 1:
        pred[X[:, dim] <= thresh] = -1
    else:
        pred[X[:, dim] >  thresh] = -1
    return pred                       # array of +1 / -1

    Start with uniform sample weights
    
        
    

    # every training point begins equally important. the weights form a
# probability distribution over the data (they sum to 1). as boosting
# proceeds, points the ensemble keeps getting wrong will grow heavier.
n = len(X)
w = np.full(n, 1.0 / n)               # uniform: every point weight 1/n
# labels y must be encoded as +1 / -1 (NOT 0/1) for the math below.
    
          
        
    
      
    3. 
      
  3. 
        
    3
    Pick the stump with the lowest WEIGHTED error
    
        
    

    # the best stump this round is the one that misclassifies the least
# WEIGHT (not the least count). heavy points dominate the search, so
# the stump is pulled toward whatever the ensemble is failing on.
def best_stump(X, y, w):
    best = None; best_err = float("inf")
    for dim in range(X.shape[1]):
        for thresh in np.unique(X[:, dim]):
            for polarity in (1, -1):
                pred = stump_predict(X, dim, thresh, polarity)
                err = w[pred != y].sum()      # weighted error
                if err < best_err:
                    best_err = err
                    best = (dim, thresh, polarity)
    return best, best_err
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Turn the error into a voting weight α
    
        
    

    # alpha is how much this stump's vote counts in the final ensemble.
# a stump with tiny error earns a big alpha; one at err = 0.5 (useless)
# earns alpha = 0; one that's WORSE than chance gets a negative alpha
# (its vote is flipped). the 1e-10 guards against a divide-by-zero.
eps = 1e-10
err = min(max(best_err, eps), 1 - eps)
alpha = 0.5 * np.log((1 - err) / err)
    
          
        
    
      
    5. 
      
  5. 
        
    5
    Reweight: grow the misclassified points, renormalise
    
        
    

    # multiply each weight by exp(-alpha * y_i * h(x_i)). when the stump is
# RIGHT (y_i*h = +1) the weight shrinks; when it's WRONG (y_i*h = -1)
# the weight grows by exp(alpha). then divide by the total Z so the
# weights sum back to 1 -> a fresh distribution the next round chases.
pred = stump_predict(X, dim, thresh, polarity)
w = w * np.exp(-alpha * y * pred)
w = w / w.sum()                       # renormalise: sum(w) == 1 again
    
          
        
    
      
    6. 
      
  6. 
        
    6
    The boosting loop: repeat for T rounds
    
        
    

    # stitch it together: each round appends one (stump, alpha) to the
# ensemble. the growing weights force each new stump onto the residual
# hard cases, so the group covers what any single stump cannot.
def adaboost(X, y, T):
    n = len(X); w = np.full(n, 1.0/n); ensemble = []
    for _ in range(T):
        (dim, thr, pol), err = best_stump(X, y, w)
        err = min(max(err, 1e-10), 1 - 1e-10)
        alpha = 0.5 * np.log((1 - err) / err)
        pred = stump_predict(X, dim, thr, pol)
        w = w * np.exp(-alpha * y * pred); w /= w.sum()
        ensemble.append((alpha, dim, thr, pol))
    return ensemble

  7. 
        
    7
    Predict: the weighted vote sign(Σ α·stump)
    
        
    

    # the strong classifier is a weighted majority vote. add up each
# stump's +1/-1 prediction scaled by its alpha; the SIGN of that sum
# is the final label. confident stumps (big alpha) swing the vote more.
def predict(ensemble, X):
    total = np.zeros(len(X))
    for alpha, dim, thr, pol in ensemble:
        total += alpha * stump_predict(X, dim, thr, pol)
    return np.sign(total)             # the additive ensemble decision
acc = (predict(ensemble, X) == y).mean()
print("train accuracy:", round(acc, 3))

    from sklearn.ensemble import AdaBoostClassifier
from sklearn.tree import DecisionTreeClassifier

# the weak learner is a depth-1 tree = exactly our stump.
# n_estimators is T (rounds); learning_rate shrinks each alpha to slow
# boosting down and fight overfitting. use cross-validation to pick both.
clf = AdaBoostClassifier(
    estimator=DecisionTreeClassifier(max_depth=1),
    n_estimators=50,
    learning_rate=1.0,

🎉 Day 21 of MachineLearningFromZero done.
AdaBoost builds a strong classifier from an army of weak decision stumps. Each round it picks the stump with the lowest weighted error, scores it with α = ½·ln((1−err)/err), then multiplies up the weight of every misclassified point and renormalises — so the next stump is forced onto the hard cases. The final model is the weighted vote sign(Σ α·stump), an additive ensemble that drives down bias round after round. Stop with cross-validation or a small learning rate before it starts fitting the noise.
