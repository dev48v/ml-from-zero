🌿 LightGBM — Same Gradient/Hessian Boosting as XGBoost, Grown to be Fast: Leaf-Wise Best-First Trees, Histogram Split-Finding, GOSS Sampling & EFB Feature Bundling

Day 40 opened up XGBoost's engine: a regularized objective, second-order gradient gᵢ and Hessian hᵢ, closed-form leaf weights and γ-gated gains. LightGBM keeps that exact math — every leaf is still w*=−G/(H+λ) and every split is scored by the same gain — and rebuilds the parts that make it fast on big data. Four ideas do the work, and all four are computed for real below. It grows trees leaf-wise (split the single leaf with the largest loss reduction, not a whole level), finds splits on a histogram of ~k bins instead of every sorted value, throws away most low-gradient rows with GOSS while amplifying the survivors, and squeezes sparse one-hot columns together with EFB.

Pick a mechanism below. Leaf-wise vs level-wise grows two trees on the same data one split at a time — leaf-wise always spends its next split where the loss drops most, so it reaches a lower training loss for the same number of leaves, but the tree goes deeper and lopsided (the overfitting risk a max_depth guard fights). Histogram bins a feature and scores the gain only at the ~k bin edges instead of the n−1 exact midpoints — drag the bin count and watch the coarse gain curve snap toward the exact one. GOSS keeps every large-gradient sample and randomly subsamples the small-gradient ones (amplified by (1−a)/b so the sums stay unbiased) — and its split-gain estimate beats plain uniform sampling on the same budget. EFB bundles features that are never nonzero together into one column. Everything is real JS math, no library.

LightGBM reuses Day 40's leaf weight and gain verbatim — so we start from those two functions and only add the four accelerators. Here is a runnable from-scratch core: the shared gain, an exact best-split, the leaf-wise grower (a priority queue on gain, capped by num_leaves and a max_depth guard) next to the level-wise one for contrast, histogram split-finding, GOSS sampling, and EFB bundling.     The shared core — same gain & leaf weight as XGBoost
    
        
    LightGBM is gradient boosting with the identical second-order objective. A leaf's optimal weight is −G/(H+λ) and a split's gain is the same structure-score improvement. Nothing here is new — it is the foundation the four tricks accelerate.
    
        
    import numpy as np

def grad_hess(y, F, loss="mse"):
    if loss == "mse":                       # l = 1/2 (y - F)^2
        return F - y, np.ones_like(F)       # g = F - y ,  h = 1
    p = 1/(1+np.exp(-F))                     # logistic
    return p - y, p*(1-p)                    # g = p - y ,  h = p(1-p)

def leaf_weight(G, H, lam):
    return -G / (H + lam)                    # closed-form, L2-shrunk

def split_gain(GL, HL, GR, HR, lam=0.0, gamma=0.0):
    sim = lambda g, h: g*g / (h + lam)
    G, H = GL+GR, HL+HR
    return 0.5*(sim(GL,HL) + sim(GR,HR) - sim(G,H)) - gamma

    Best split on a node — the gain each candidate leaf carries
    
        
    For every leaf we score its best possible split by sweeping thresholds. Leaf-wise growth needs this gain as the priority key: the leaf with the largest gain gets split next.
    
        
    def best_split(g, h, x, lam=0.0):
    order = np.argsort(x); g, h, x = g[order], h[order], x[order]
    G, H = g.sum(), h.sum()
    GL = HL = 0.0
    best = (-1.0, None)                       # (gain, threshold)
    for i in range(len(x) - 1):
        GL += g[i]; HL += h[i]
        if x[i] == x[i+1]:
            continue
        gain = split_gain(GL, HL, G-GL, H-HL, lam)
        if gain > best[0]:
            best = (gain, 0.5*(x[i]+x[i+1]))
    return best                                # gain, threshold (or None)

    Leaf-wise growth — split the best leaf, capped by num_leaves & max_depth
    
        
    This is LightGBM's signature. Keep a priority queue of leaves keyed by their best-split gain; pop the max, split it, push its two children, repeat until num_leaves is hit. The max_depth guard skips leaves that are already too deep — that is what stops best-first growth from overfitting into a thin, deep tree.
    
        
    import heapq
def grow_leafwise(g, h, x, num_leaves=31, max_depth=-1, lam=0.0):
    root = {"idx": np.arange(len(x)), "depth": 0}
    def push(pq, leaf):
        gain, thr = best_split(g[leaf["idx"]], h[leaf["idx"]], x[leaf["idx"]], lam)
        deep = max_depth >= 0 and leaf["depth"] >= max_depth
        if thr is not None and gain > 0 and not deep:
            heapq.heappush(pq, (-gain, id(leaf), leaf, gain, thr))  # -gain = max-heap
    pq = []; leaves = [root]; push(pq, root)
    while pq and len(leaves) < num_leaves:
        _, _, leaf, gain, thr = heapq.heappop(pq)        # the single best leaf
        li = leaf["idx"][x[leaf["idx"]] <  thr]
        ri = leaf["idx"][x[leaf["idx"]] >= thr]
        L = {"idx": li, "depth": leaf["depth"]+1}
        R = {"idx": ri, "depth": leaf["depth"]+1}
        leaves.remove(leaf); leaves += [L, R]
        push(pq, L); push(pq, R)
    return leaves      # deeper, imbalanced -> lower train loss per leaf

    Level-wise growth — XGBoost's default, for contrast
    
        
    Same splits, different order: a plain FIFO queue splits leaves breadth-first, filling each depth level before descending. The tree stays balanced, but the same leaf budget buys a higher training loss because splits are spent on regions that barely needed them.
    
        
    from collections import deque
def grow_levelwise(g, h, x, num_leaves=31, lam=0.0):
    root = {"idx": np.arange(len(x)), "depth": 0}
    q = deque([root]); leaves = [root]
    while q and len(leaves) < num_leaves:
        leaf = q.popleft()                                # FIFO = breadth-first
        gain, thr = best_split(g[leaf["idx"]], h[leaf["idx"]], x[leaf["idx"]], lam)
        if thr is None or gain <= 0:
            continue
        li = leaf["idx"][x[leaf["idx"]] <  thr]
        ri = leaf["idx"][x[leaf["idx"]] >= thr]
        L = {"idx": li, "depth": leaf["depth"]+1}
        R = {"idx": ri, "depth": leaf["depth"]+1}
        leaves.remove(leaf); leaves += [L, R]; q += [L, R]
    return leaves

    Histogram split-finding — bin once, sum per bin, sweep the edges
    
        
    Instead of sorting a feature at every node, bin it once into ~255 buckets and accumulate G,H per bin. A split search is then a single cumulative pass over the bin edges — O(#bins) instead of O(#rows). The subtraction trick builds a child histogram as parent minus sibling, for free.
    
        
    def hist_best_split(g, h, x, max_bins=255, lam=0.0):
    edges = np.quantile(x, np.linspace(0, 1, max_bins+1))   # bin ONCE
    edges = np.unique(edges)
    b = np.clip(np.digitize(x, edges[1:-1]), 0, len(edges)-2)
    Gb = np.bincount(b, weights=g, minlength=len(edges)-1)  # per-bin sums
    Hb = np.bincount(b, weights=h, minlength=len(edges)-1)
    G, H = Gb.sum(), Hb.sum()
    GL = HL = 0.0; best = (-1.0, None)
    for k in range(len(Gb) - 1):                            # only k-1 edges
        GL += Gb[k]; HL += Hb[k]
        gain = split_gain(GL, HL, G-GL, H-HL, lam)
        if gain > best[0]:
            best = (gain, edges[k+1])
    return best        # child_hist = parent_hist - sibling_hist  (free)

    GOSS & EFB — sample rows, bundle columns
    
        
    GOSS keeps every large-gradient row and randomly subsamples the small-gradient rest, amplifying those survivors by (1−a)/b so the gradient sums stay unbiased. EFB packs mutually-exclusive sparse features into one column by greedily colouring a conflict graph and offsetting value ranges. Together they shrink the data both ways before a single tree is grown.
    
        
    # ---- GOSS: gradient-based one-side sampling ----
def goss_subset(g, h, a=0.2, b=0.2):
    n = len(g); top = int(a*n); rest = int(b*n)
    order = np.argsort(-np.abs(g))            # big |gradient| first
    keep  = order[:top]                       # keep ALL large-gradient rows
    small = order[top:]
    samp  = np.random.choice(small, rest, replace=False)
    amp   = (1 - a) / b                        # up-weight the survivors
    idx   = np.concatenate([keep, samp])
    w     = np.ones(n); w[samp] = amp          # unbiased gradient sums
    return idx, w                              # gain uses g*w, h*w on idx

# ---- EFB: exclusive feature bundling ----
def efb_bundle(X, max_conflict=0):
    F = X.shape[1]
    conflict = [[int(np.sum((X[:,i]!=0) & (X[:,j]!=0)))       # shared nonzeros
                 for j in range(F)] for i in range(F)]
    color = [-1]*F                              # greedy graph colouring
    for i in range(F):
        used = {color[j] for j in range(F)
                if color[j] >= 0 and conflict[i][j] > max_conflict}
        c = 0
        while c in used: c += 1
        color[i] = c
    return color        # features sharing a colour merge into one bundle
