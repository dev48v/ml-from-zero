🌲 Gradient Boosting — Many Weak Trees, Fixing Each Other

Watch boosting bend a flat line into the curve, one tree at a time

Start with a constant guess — the mean of all the points. Each round fits a tiny regression tree to the residuals (the errors still left), then adds a small slice of it to the running prediction, scaled by the learning rate. The sky curve is the ensemble so far; the rose bars are the residuals. Add trees and watch the curve bend to the data while the bars shrink.

Boosting builds many weak trees sequentially — each one only learns the mistakes the ensemble has left over. That's the opposite of a random forest, which grows many full trees independently in parallel and averages them.

🔨 Build it yourself

    
Gradient boosting from scratch: start the prediction at the mean, then loop — compute the residuals, fit a small tree to them, add learning-rate × tree to the running prediction, repeat. A handful of honest steps and you've built the algorithm behind XGBoost.

    
    
      
  1. 
        
    1
    The data + a weak learner (a shallow tree)
    
        
    

    // regression: predict a continuous y from features X.
const X = [[0.1],[0.4],[0.6],[0.9],[1.2],[1.5],[1.8],[2.1]];
const y = [0.2, 0.9, 1.1, 0.4,-0.3,-0.8,-0.5, 0.6];

// a "weak learner" = a tiny tree, depth 1-3. on its own it's
// barely better than guessing — that's the point. boosting
// turns a crowd of these weaklings into one strong predictor.
    
          
        
    
      
    2. 
      
  2. 
        
    2
    Initialise the prediction = the mean
    
        
    

    // the dumbest possible model: predict the average for everyone.
// for squared-error loss the mean is provably the best constant.
// every tree from here on only has to fix what this leaves wrong.
const n = y.length;
const base = y.reduce((s, v) => s + v, 0) / n;   // F0(x) = mean
let pred = Array(n).fill(base);                  // running prediction
const lr = 0.3;                                  // learning rate (shrinkage)
const trees = [];                                // the ensemble we build up
    
          
        
    
      
    3. 
      
  3. 
        
    3
    Compute the residuals (what's still wrong)
    
        
    

    // residual = the error still left after the current ensemble.
// for squared-error loss this is exactly the negative gradient,
// which is why the method is called *gradient* boosting.
// the NEXT tree's whole job is to predict these residuals.
function residuals(y, pred) {
  return y.map((yi, i) => yi - pred[i]);   // r_i = y_i - F(x_i)
}
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Fit a small tree to the residuals
    
        
    

    // fit a shallow regression tree to predict r from X. a tree
// splits the feature into regions and predicts a constant
// (the mean residual) inside each region. depth 2 ≈ 4 leaves.
function fitStump(X, r) {
  // pick the split that most reduces the residual variance,
  // then store the mean residual on each side as the leaf value.
  const best = bestSplit(X, r);              // {feature, threshold}
  const left = mean(r.filter((_, i) => X[i][best.feature] <  best.threshold));
  const right= mean(r.filter((_, i) => X[i][best.feature] >= best.threshold));
  return { ...best, left, right };
}
    
          
        
    
      
    5. 
      
  5. 
        
    5
    Add learning-rate × tree to the prediction
    
        
    

    // don't trust any one tree fully — take only a small step toward
// it. the learning rate (shrinkage) keeps each correction modest,
// so the ensemble inches toward the answer and generalizes better.
function predictTree(tree, x) {
  return x[tree.feature] < tree.threshold ? tree.left : tree.right;
}
function addTree(tree) {
  trees.push(tree);
  for (let i = 0; i < n; i++)
    pred[i] += lr * predictTree(tree, X[i]);   // F = F + lr · tree
}
    
          
        
    
      
    6. 
      
  6. 
        
    6
    Loop — and stop early
    
        
    

    // repeat: residuals → fit tree → add it. each round shrinks the
// error a little more. watch a VALIDATION error, not just training:
// when it stops dropping, stop adding trees (early stopping) —
// otherwise the ensemble starts memorising noise and overfits.
const MAX_TREES = 200;
for (let m = 0; m < MAX_TREES; m++) {
  const r = residuals(y, pred);
  addTree(fitStump(X, r));
  const trainMSE = mean(r.map(e => e * e));
  // if validationMSE has not improved for K rounds → break;
}
    
          
        
    
      
    7. 
      
  7. 
        
    7
    Predict = mean + sum of all the steps
    
        
    

    // to score a NEW point, start at the base mean and add every
// tree's contribution, each scaled by the same learning rate.
// the final model is just one big sum of tiny corrections.
function predict(x) {
  let f = base;
  for (const tree of trees) f += lr * predictTree(tree, x);
  return f;
}
// F(x) = F0 + lr·tree1(x) + lr·tree2(x) + ... + lr·treeM(x)

    In the real world: XGBoost
    
        
    

    from xgboost import XGBRegressor

model = XGBRegressor(
    n_estimators=500,        # number of boosting rounds (trees)
    learning_rate=0.05,      # shrinkage — small + many trees generalizes
    max_depth=3,             # keep each tree weak/shallow
    subsample=0.8,           # row sampling = regularization
    early_stopping_rounds=20 # stop when validation stops improving
)
model.fit(X_train, y_train, eval_set=[(X_val, y_val)])
# LightGBM and CatBoost are the same idea, tuned for speed/categories.
# Gradient boosting is the go-to winner for TABULAR data.
