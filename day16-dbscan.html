🌌 DBSCAN — Clustering by Density (with Noise)

Watch density carve out clusters of any shape — and flag the loners as noise

DBSCAN never asks you how many clusters there are. It just grows them outward from dense regions. A point is a core point if at least minPts neighbours sit within radius eps; clusters form by chaining core points together, sweeping in their border neighbours, and leaving the stragglers as noise. Pick the two-moons or rings dataset — shapes K-Means could never split — drag eps and minPts, and watch the clusters and the gray ✕ noise points emerge from the data itself.

🔨 Build it yourself

    
DBSCAN from scratch: write a region query that finds every neighbour within eps, label each point core / border / noise, then grow each cluster by a breadth-first expansion over density-connected core points. No K, no centroids — just two knobs and a flood fill.

    1
    The data + the two knobs
    
        
    

    // points live in 2-D; no labels — clustering is unsupervised.
const P = [[0.10,0.20],[0.12,0.18],[0.14,0.22],   // a dense clump
           [0.80,0.78],[0.82,0.81],[0.79,0.83],   // another clump
           [0.50,0.95]];                           // a lonely outlier

// DBSCAN has exactly two hyper-parameters — and that's all:
const eps    = 0.08;   // neighbourhood radius
const minPts = 3;      // neighbours-within-eps needed to be "dense"
// you do NOT pass the number of clusters — it emerges from the data.

  2. 
        
    2
    The region query — find the eps-neighbours
    
        
    

    // the heart of DBSCAN: given a point, return every OTHER point
// (its own index included) that sits within distance eps.
function dist(a, b) {
  return Math.hypot(a[0] - b[0], a[1] - b[1]);   // Euclidean
}
function regionQuery(P, i, eps) {
  const neighbours = [];
  for (let j = 0; j < P.length; j++)
    if (dist(P[i], P[j]) <= eps) neighbours.push(j);
  return neighbours;                              // includes i itself
}

  3. 
        
    3
    Label core points (the dense ones)
    
        
    

    // a point is a CORE point if its eps-neighbourhood is crowded:
// at least minPts points (counting itself) live within eps.
// core points are the seeds clusters will grow out from.
const isCore = P.map((_, i) =>
  regionQuery(P, i, eps).length >= minPts
);
// border points (in a core's neighbourhood but not core themselves)
// and noise points get sorted out during expansion, below.
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Scan every point, seed a cluster from each core
    
        
    

    // labels: -1 = noise (default), 0,1,2,… = cluster id.
const labels = Array(P.length).fill(undefined);   // undefined = unvisited
let clusterId = 0;

for (let i = 0; i < P.length; i++) {
  if (labels[i] !== undefined) continue;          // already handled
  const neigh = regionQuery(P, i, eps);
  if (neigh.length < minPts) { labels[i] = -1; continue; }  // not core → noise (for now)
  expandCluster(P, labels, i, neigh, clusterId, eps, minPts); // grow a cluster
  clusterId++;
}
    
          
        
    
      
    5. 
      
  5. 
        
    5
    Expand the cluster with a BFS flood-fill
    
        
    

    // from a core seed, flood outward: visit every neighbour, claim it,
// and if a neighbour is ALSO core, add ITS neighbours to the queue.
// this chains density-reachable core points into one cluster — and
// is exactly why DBSCAN can trace long, curved, arbitrary shapes.
function expandCluster(P, labels, i, neigh, cid, eps, minPts) {
  labels[i] = cid;
  const queue = [...neigh];
  for (let q = 0; q < queue.length; q++) {
    const j = queue[q];
    if (labels[j] === -1) labels[j] = cid;        // noise → border of THIS cluster
    if (labels[j] !== undefined) continue;        // already in a cluster
    labels[j] = cid;
    const jn = regionQuery(P, j, eps);
    if (jn.length >= minPts) queue.push(...jn);   // j is core → keep expanding
  }                                                // else j is a border point, stop
}

    // after the scan, every point carries a label:
//   -1        → noise / outlier  (belongs to no cluster)
//   0,1,2,…   → which cluster it landed in
const noise    = labels.filter(l => l === -1).length;
const nClusters = new Set(labels.filter(l => l >= 0)).size;

console.log(`found ${nClusters} clusters, ${noise} noise points`);
// the COUNT of clusters was never specified — it fell out of the data,
// and points that never joined a dense region are honestly flagged noise.

    Choosing eps — the k-distance elbow
    
        
    

    # a principled way to pick eps: for every point, measure the distance
# to its k-th nearest neighbour (k ≈ minPts), sort those distances,
# and plot them. the "elbow" — where the curve bends sharply up —
# is a good eps: below it points are dense, above it they're sparse.
from sklearn.neighbors import NearestNeighbors
import numpy as np

k = minPts
nbrs = NearestNeighbors(n_neighbors=k).fit(X)
d, _ = nbrs.kneighbors(X)
kdist = np.sort(d[:, -1])      # distance to k-th neighbour, ascending
# plot kdist → read eps off the elbow. rule of thumb: minPts ≈ 2 * dims.

    from sklearn.cluster import DBSCAN

db = DBSCAN(eps=0.08, min_samples=4).fit(X)
labels = db.labels_            # -1 = noise, 0,1,2,… = cluster ids
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
core_mask = db.core_sample_indices_   # which points were core

# scikit-learn uses a KD-/ball-tree so region queries are fast.
# DBSCAN shines for spatial/geo data and anomaly detection; when
# clusters have very different densities, try HDBSCAN instead.
