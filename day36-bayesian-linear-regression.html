🔮 Bayesian Linear Regression — Not One Line, a Distribution of Lines With Honest Error Bars

Click anywhere on the chart to drop data points. Ordinary least-squares would draw one best-fit line and stop. Bayesian regression keeps a whole distribution over lines — a mean line, a cloud of plausible lines sampled from the posterior, and a shaded ±σ uncertainty band that balloons out where you have no data and tightens as points pile in. Every number is solved in closed form in your browser.

The model is a straight line y = w₀ + w₁·x with Gaussian noise of precision β (so noise variance is 1/β). We start from a Gaussian prior on the weights, w ~ N(0, α⁻¹I) — prior precision α pulls the line gently toward flat. Multiplying prior × likelihood gives a Gaussian posterior with precision Aₙ = αI + βΦᵀΦ, covariance Σₙ = Aₙ⁻¹ and mean mₙ = β·Σₙ·Φᵀy. A prediction at input x* is itself Gaussian: mean mₙᵀφ(x*) and variance 1/β + φ(x*)ᵀΣₙφ(x*) — the second term is the model's own uncertainty, and it grows quadratically as you move away from where the data lives. Move the sliders, drop points, and watch the band breathe.

🔨 Build it yourself

    
scikit-learn ships this as BayesianRidge — it even learns the prior strength α and the noise precision β from the data for you (via the marginal likelihood / evidence). Below is the whole thing: fit it, get predictions with error bars, then the from-scratch closed-form posterior and predictive that produced every band and sampled line you just dragged around.

    
    
      
  1. 
        
    1
    The design matrix Φ — turn inputs into features
    
        
    A straight line is just two basis functions: the constant 1 (the intercept) and x (the slope). Stack them into a matrix Φ with one row per data point. Swap in polynomial or Gaussian basis functions later and the exact same posterior maths gives you curved fits with the same honest uncertainty.
    
        
    import numpy as np

x = np.array([-4., -2.5, -1., 0.5, 3.])   # your inputs
y = np.array([-2.1, -1.0, 0.2, 0.6, 2.4]) # your targets

# basis functions phi(x) = [1, x]  ->  design matrix, one row per point
Phi = np.c_[np.ones_like(x), x]            # shape (N, 2)
    
      
    2. 
      
  2. 
        
    2
    The prior and the likelihood
    
        
    Before seeing data we say the weights are small: w ~ N(0, α⁻¹I). The likelihood says each target is the line plus Gaussian noise of precision β: y ~ N(Φw, β⁻¹I). These two knobs are the α and β sliders in the demo — the entire model lives in them.
    
        
    alpha = 2.0     # prior precision  -> prior variance is 1/alpha
beta  = 6.0     # noise precision  -> noise std is 1/sqrt(beta)

# prior:      p(w)   = N(w | 0, (1/alpha) I)
# likelihood: p(y|w) = N(y | Phi @ w, (1/beta) I)
    
      
    3. 
      
  3. 
        
    3
    The posterior — closed form (the demo engine)
    
        
    A Gaussian prior times a Gaussian likelihood is again Gaussian (conjugacy), so no sampling is needed. The posterior precision is Aₙ = αI + βΦᵀΦ, its covariance is Σₙ = Aₙ⁻¹, and its mean is mₙ = β·Σₙ·Φᵀy. This is exactly what runs on every slider move.
    
        
    A = alpha * np.eye(Phi.shape[1]) + beta * Phi.T @ Phi   # posterior precision
S = np.linalg.inv(A)                                    # posterior covariance
m = beta * S @ Phi.T @ y                                # posterior mean weights

print("mean weights [w0, w1]:", m)
print("weight std devs:", np.sqrt(np.diag(S)))
    
      
    4. 
      
  4. 
        
    4
    The predictive distribution — mean ± the band
    
        
    For a new input x* the prediction is Gaussian. Its mean is mₙᵀφ(x*); its variance is 1/β + φ(x*)ᵀΣₙφ(x*) — noise floor plus model uncertainty. That second term is a parabola in x*, so the band is narrowest near your data and flares outward. Adding points shrinks Σₙ, so the band thins toward the noise floor.
    
        
    def predict(xs):
    P    = np.c_[np.ones_like(xs), xs]                 # basis at test points
    mean = P @ m                                       # posterior mean line
    var  = 1.0/beta + np.einsum('ij,jk,ik->i', P, S, P)  # 1/beta + phiᵀ S phi
    return mean, np.sqrt(var)                          # <- std = the band

xs = np.linspace(-5, 5, 200)
mean, std = predict(xs)
band_low, band_high = mean - std, mean + std           # +/- 1 sigma band
    
      
    5. 
      
  5. 
        
    5
    Sample whole lines from the posterior
    
        
    The band summarises the uncertainty, but drawing actual lines makes it visceral. Sample weight vectors from N(mₙ, Σₙ) and plot each as a line — where they fan apart is where the model is unsure. These are the grey lines in the demo (done there with a 2×2 Cholesky factor).
    
        
    W = np.random.multivariate_normal(m, S, size=20)   # 20 plausible weight vectors
xs = np.linspace(-5, 5, 2)
for w0, w1 in W:
    plt.plot(xs, w0 + w1 * xs, color="grey", alpha=0.3)   # one sampled line each
plt.plot(xs, m[0] + m[1] * xs, color="dodgerblue", lw=2)  # posterior mean

    The one-liner: scikit-learn learns α and β for you
    
        
    BayesianRidge does everything above and estimates α and β from the data by maximising the evidence — no hand-tuning. Ask for return_std=True and you get the error bars for free. Reach for it when a prediction interval matters (forecasting, active learning, safety) and for plain point predictions on small/wide data where it self-regularises.
    
        
    from sklearn.linear_model import BayesianRidge

model = BayesianRidge()                 # learns alpha & beta via the evidence
model.fit(X, y)                         # X shape (N, n_features)
mean, std = model.predict(X_test, return_std=True)   # predictions WITH error bars

# when to reach for it:
# ridge regression  = the MAP point estimate of this model (alpha = lambda)
# least squares     = the limit alpha -> 0 (flat prior): one line, no error bars
# full Bayes (here) = the whole posterior -> calibrated uncertainty everywhere
# want curves + uncertainty of any shape? swap the basis -> Gaussian Processes

The predictive variance 1/β + φᵀΣₙφ gave you a band that widens where data is sparse and tightens as points arrive, never dropping below the noise floor 1/√β.
