⚖️ Bias-Variance — The Tradeoff Behind Over/Underfitting

Slide the model complexity and watch underfit turn into overfit

The faint gray curve on the left is the true function; the dots are noisy samples of it. Drag the complexity slider (polynomial degree 1→12) and we actually fit that model to the data — the bright sky curve is the fit. We also fit on several resampled datasets and overlay those fits (thin lines): at low complexity they all look the same but miss the curve (high bias / underfit); at high complexity they wiggle wildly and disagree (high variance / overfit). The right chart plots real measured error vs complexity: training error falls forever, but test error is U-shaped — high when too simple, lowest at the sweet spot, climbing again when too complex. The dot marks your current degree. Total error = bias² + variance + irreducible noise: too simple → high bias, too complex → high variance, and the best model is the dip in between.

🔨 Build it yourself

    
Reproduce the U-shaped curve from scratch: generate noisy data from a known true function, fit polynomial models of increasing degree, measure training error and held-out test error for each, plot both against complexity, and pick the degree with the lowest test error. You'll watch train error fall forever while test error dives then climbs — the bias-variance tradeoff, made of real numbers you computed.

    
    
      
  1. 
        
    1
    Generate noisy data from a known true curve
    
        
    

    # the "true" signal is a smooth function we secretly know.
# real samples = true(x) + irreducible noise. that noise is the
# error floor no model can ever beat — it's not bias or variance.
import numpy as np
rng = np.random.default_rng(0)
def true_f(x): return np.sin(2*np.pi*x) * 0.7      # the smooth signal
x = np.sort(rng.uniform(0, 1, 60))
y = true_f(x) + rng.normal(0, 0.18, x.shape)       # + irreducible noise

  2. 
        
    2
    Split into train and held-out test
    
        
    

    # test error is the only honest gauge of overfitting, so carve out
# a chunk the models never train on. train error alone always drops
# with complexity — it can't tell you when you've started overfitting.
from sklearn.model_selection import train_test_split
x_tr, x_te, y_tr, y_te = train_test_split(x, y, test_size=0.35,
                                          random_state=0)
    
          
        
    
      
    3. 
      
  3. 
        
    3
    Fit polynomials of increasing degree
    
        
    

    # complexity = polynomial degree. degree 1 is a line (high bias);
# a high degree can snake through every training point (high variance).
# np.polyfit fits degree-d least-squares; np.polyval evaluates it.
def fit_poly(xt, yt, degree):
    return np.polyfit(xt, yt, degree)        # the model's coefficients
def predict(coef, xq):
    return np.polyval(coef, xq)
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Measure train vs held-out error at each degree
    
        
    

    # for every candidate degree, fit on TRAIN only, then score both
# the training points and the untouched test points. collect both
# curves — this is the raw material of the bias-variance picture.
def mse(a, b): return np.mean((a - b) ** 2)
degrees = range(1, 13)
train_err, test_err = [], []
for d in degrees:
    coef = fit_poly(x_tr, y_tr, d)
    train_err.append(mse(predict(coef, x_tr), y_tr))   # always falls
    test_err.append(mse(predict(coef, x_te), y_te))    # U-shaped

  5. 
        
    5
    Plot both curves — see the U
    
        
    

    # train error sliding down while test error dives then climbs is the
# canonical bias-variance plot. the gap between the two curves widens
# as variance grows: a big train/test gap is the signature of overfit.
import matplotlib.pyplot as plt
plt.plot(degrees, train_err, "o-", label="train (always drops)")
plt.plot(degrees, test_err,  "o-", label="test (U-shaped)")
plt.xlabel("model complexity (degree)"); plt.ylabel("MSE"); plt.legend()
plt.show()

    Pick the min-test-error complexity
    
        
    

    # the sweet spot is simply the bottom of the U: the degree whose
# held-out error is smallest. that's the best bias-variance balance
# for this data. (in practice use CV instead of one split — Day 18.)
best_i = int(np.argmin(test_err))
best_degree = list(degrees)[best_i]
print("sweet spot: degree", best_degree,
      "test MSE", round(test_err[best_i], 4))

    Estimate variance by refitting on resamples
    
        
    

    # variance = how much the fit JUMPS when the training data changes.
# refit the same degree on many resampled datasets and look at the
# spread of predictions at each x. high degree -> huge spread.
def fit_variance(degree, n_sets=40, grid=np.linspace(0,1,100)):
    preds = []
    for _ in range(n_sets):
        xs = np.sort(rng.uniform(0,1,60))
        ys = true_f(xs) + rng.normal(0,0.18,xs.shape)   # a fresh draw
        preds.append(predict(fit_poly(xs, ys, degree), grid))
    return np.var(np.array(preds), axis=0).mean()       # avg variance
print(fit_variance(1), fit_variance(11))   # low vs high

    In the real world: validation curves & regularization
    
        
    

    from sklearn.model_selection import validation_curve
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import Ridge

# scikit-learn draws the whole train-vs-CV curve in one call
pipe = make_pipeline(PolynomialFeatures(), Ridge(alpha=1.0))
tr, va = validation_curve(pipe, x.reshape(-1,1), y,
            param_name="polynomialfeatures__degree",
            param_range=range(1,13), cv=5, scoring="neg_mean_squared_error")
# levers that move the tradeoff: less complexity / more data / a
# bigger Ridge alpha all CUT variance; richer features cut bias.

🎉 Day 19 of MachineLearningFromZero done.
Every model's error splits into bias² + variance + irreducible noise. Too simple means high bias and underfitting; too complex means high variance and overfitting; lowering one tends to raise the other. Training error keeps falling, but the U-shaped test error reveals the sweet spot — find it with validation or cross-validation, and shift it with more data, regularization, or smarter features. Next up: regularization (Ridge & Lasso) — the dial that tames variance directly.
