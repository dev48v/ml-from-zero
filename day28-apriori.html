🛒 Apriori & Association Rules — Mine "People Who Bought X Also Bought Y"

Nobody told it diapers go with beer. It found out from the receipts.

Below is a tiny stack of shopping baskets — each row is one checkout, each blue cell an item that was in it. Apriori reads nothing but co-occurrence and answers two questions: which sets of items show up together often enough to matter (frequent itemsets), and which of those become trustworthy if-then rules (X → Y). It works in levels — first single items, then pairs, then triples — and its one clever trick is the Apriori principle: if a set is rare, every superset of it must be rare too, so whole branches of candidates get pruned before they are ever counted. Slide min-support and min-confidence, toggle items in the baskets, and watch the frequent itemsets grow level-by-level, the pruned candidates get culled, and the rules re-sort by lift. The classic {diapers} → {beer} rule is highlighted when it survives.

ASSOCIATION RULES — sorted by lift · lift>1 positive · ≈1 independent · <1 negative
Support, confidence, lift — the three numbers: support(X) is how often X appears (popularity); confidence(X→Y) = support(X∪Y)/support(X) is how often Y shows up given X (reliability); lift = confidence / support(Y) asks whether X actually makes Y more likely than chance. Confidence alone lies: {anything}→{bread} looks strong simply because almost every basket has bread — that is the base-rate trap. Lift corrects for it: lift>1 means a genuine positive association, =1 means independent, <1 means buying X actually makes Y less likely. That is why the rules table is ranked by lift, not confidence.

🔨 Build it yourself

    
One-hot the baskets, count support, keep the frequent singletons, then grow candidates level-by-level with the join + downward-closure prune that is the whole point of Apriori — and finally read out the rules with confidence and lift. All in vanilla JS, then the real library version in a handful of lines.

    The baskets & support
    
        
    Everything starts from a list of transactions — each a set of item names. Support of an itemset is just the fraction of baskets that contain all of its items. It is the one primitive the whole algorithm counts.
    
        
    const TX = [
  ["milk","bread","butter"],
  ["beer","diapers","chips"],
  ["milk","bread","beer","diapers","chips"],
  // ... one array per checkout
];
const N = TX.length;
const sets = TX.map(t => new Set(t));
const support = items =>                 // fraction of baskets containing ALL items
  sets.filter(s => items.every(it => s.has(it))).length / N;

    Level 1: frequent single items
    
        
    Score every individual item and drop the ones below minSup. This first cull is what makes everything after it cheap — a rare item can never be part of a frequent pair, so it is gone for good.
    
        
    const items = [...new Set(TX.flat())];              // the item universe
function frequentSingletons(minSup){
  return items
    .map(it => ({ set:[it], sup:support([it]) }))
    .filter(c => c.sup >= minSup);                   // min-support prune
}
let L = frequentSingletons(minSup);                 // L1

    3
    Candidate generation: join
    
        
    Keep items in a fixed canonical order. To build (k+1)-candidates, join two frequent k-itemsets that share their first k−1 items and differ only in the last — that produces each candidate exactly once, no duplicates.
    
        
    function join(Lk, k){                               // Lk = frequent k-itemsets (sorted)
  const out = [];
  for (let i=0;i<Lk.length;i++)
    for (let j=i+1;j<Lk.length;j++){
      const a=Lk[i].set, b=Lk[j].set;
      let share=true;
      for (let x=0;x<k-1;x++) if (a[x]!==b[x]) share=false;   // first k-1 match
      if (share && rank(a[k-1]) < rank(b[k-1]))                // keep last ordered
        out.push([...a.slice(0,k-1), a[k-1], b[k-1]]);
    }
  return out;                                       // C(k+1)
}

  4. 
        
    4
    The Apriori prune (downward closure)
    
        
    Before spending a single scan counting a candidate, check that every one of its k-subsets was frequent. If even one wasn't, the candidate cannot possibly be frequent — kill it now. This is the Apriori principle, and it is what turns an exponential search into a tractable one.
    
        
    function prune(cands, prevKeys){                     // prevKeys = Set of frequent k-itemset keys
  return cands.filter(c =>
    c.every((_, drop) =>                             // every (k)-subset...
      prevKeys.has(key(c.filter((_, ix) => ix !== drop)))   // ...must be frequent
    ));                                             // else: never even counted
}
function apriori(minSup){
  let L = frequentSingletons(minSup), all = [...L], k = 1;
  while (L.length){
    const prevKeys = new Set(L.map(x => key(x.set)));
    const cands = prune(join(L, k), prevKeys);       // JOIN then PRUNE
    L = cands.map(set => ({set, sup:support(set)}))
             .filter(c => c.sup >= minSup);          // then count + min-support
    all.push(...L); k++;
  }
  return all;                                        // every frequent itemset
}

    5
    Rules: confidence & lift
    
        
    Frequent itemsets are only the raw material. For every itemset of size ≥2, split it into an antecedent X and consequent Y in every way, then score the rule. Keep the ones above minConf and rank by lift, which is the honest measure of association.
    
        
    function rules(freq, minConf){
  const out = [];
  for (const {set, sup} of freq){
    if (set.length < 2) continue;
    for (const [X, Y] of properSplits(set)){         // every non-empty X | Y
      const conf = sup / support(X);                 // P(Y | X)
      const lift = conf / support(Y);                // vs P(Y): >1 = positive
      if (conf >= minConf) out.push({X, Y, sup, conf, lift});
    }
  }
  return out.sort((a,b) => b.lift - a.lift);          // best associations first
}

    In the real world — mlxtend
    
        
    Nobody hand-rolls this at scale. mlxtend gives you Apriori and rule generation in a few lines over a one-hot DataFrame; for big, dense data swap in fpgrowth (same API, no candidate generation) which is Apriori's faster successor.
    
        
    # pip install mlxtend
import pandas as pd
from mlxtend.preprocessing import TransactionEncoder
from mlxtend.frequent_patterns import apriori, association_rules

te = TransactionEncoder()
onehot = pd.DataFrame(te.fit_transform(TX), columns=te.columns_)   # one-hot baskets

freq  = apriori(onehot, min_support=0.3, use_colnames=True)        # frequent itemsets
rules = association_rules(freq, metric="confidence", min_threshold=0.6)
print(rules.sort_values("lift", ascending=False)
           [["antecedents","consequents","support","confidence","lift"]])
