🔦 Bayesian Optimization — Find the Peak of an Expensive Black-Box in a Handful of Evaluations

The objective is hidden. Press "Sample next" and watch it get found in a handful of evaluations. Day 37 built a Gaussian Process that gives a calibrated uncertainty band. Today we turn that band into a decision. When every evaluation of f(x) is expensive — a training run, a lab experiment, an A/B test — you can't afford a grid. Bayesian Optimization fits the GP as a cheap surrogate, then an acquisition function reads its mean and band to score every candidate x by how promising it is. Evaluate the true function at the acquisition's argmax, add the point, refit — repeat. In a dozen steps it homes in on the peak while a random search is still wandering.

Two acquisition functions, switchable below. Expected Improvement EI(x) = 𝔼[max(0, f(x) − f⁺)] scores the expected amount you'd beat the best value seen so far f⁺; it is large where the mean is high (exploit) or the band is wide (explore), and collapses to zero at points you've already measured. Upper Confidence Bound UCB(x) = μ(x) + κ·σ(x) is an optimist's score with one dial: small κ exploits the current best, large κ chases uncertainty. Every mean, band and acquisition curve on this page is computed live in your browser with the same Cholesky-based GP you dragged around yesterday.

🔨 Build it yourself

    
scikit-optimize ships this as gp_minimize and BoTorch/Ax do it on the GPU — but the whole loop is six short pieces you already have most of. The surrogate is literally Day 37's Gaussian Process; all we add is a cheap acquisition function and a five-line loop that fits, maximises the acquisition, evaluates the true function once, and repeats.

    
    
      
  1. 
        
    1
    The expensive black-box and a few seed points
    
        
    Bayesian Optimization is for functions that are expensive to evaluate: a model training run, a simulation, a lab experiment. You don't know the formula and you can only afford a handful of calls. Start with two or three seed evaluations to give the surrogate something to fit.
    
        
    import numpy as np

# an EXPENSIVE black-box: each call costs real money/time, formula unknown to us
def objective(x):
    return 1.7*np.exp(-(x-7)**2/1.6) + 1.15*np.exp(-(x-2.6)**2/2.5) - 0.15   # hidden

bounds = (0.0, 10.0)
X = np.array([1.6, 8.4])                    # a few seed evaluations
y = np.array([objective(x) for x in X])     # the only truth we have

    2
    The surrogate — reuse Day 37's Gaussian Process
    
        
    The surrogate is a cheap stand-in for the real function that we can query for free. A GP is perfect: it returns a posterior mean and a calibrated ±σ band at every input. This is exactly yesterday's engine — kernel, three matrices, one Cholesky solve.
    
        
    def rbf(a, b, sf2=1.3, ell=1.2):
    d = a[:, None] - b[None, :]
    return sf2 * np.exp(-d**2 / (2*ell**2))

def gp_posterior(X, y, Xs, noise=1e-3):
    K   = rbf(X, X) + noise*np.eye(len(X))       # train-train
    Ks  = rbf(Xs, X)                             # test-train
    Kss = rbf(Xs, Xs)                            # test-test
    L     = np.linalg.cholesky(K)                # K = L Lᵀ  (Day 37)
    alpha = np.linalg.solve(L.T, np.linalg.solve(L, y))
    mean  = Ks @ alpha                           # posterior mean
    v     = np.linalg.solve(L, Ks.T)
    std   = np.sqrt(np.clip(np.diag(Kss) - np.sum(v*v, 0), 0, None))
    return mean, std                             # <-- the surrogate's belief

    3
    Expected Improvement — the acquisition function
    
        
    An acquisition function scores every candidate x by how worth sampling it is, folding mean and uncertainty into one number. Expected Improvement asks: how much do we expect to beat the best value so far, f⁺? It rewards a high mean (exploit) and a wide band (explore) at once, and goes to zero at points already measured — so it never wastes an evaluation.
    
        
    from scipy.stats import norm

def expected_improvement(mean, std, f_best, xi=0.01):
    std = np.maximum(std, 1e-9)
    imp = mean - f_best - xi          # xi nudges toward more exploration
    z   = imp / std
    return imp*norm.cdf(z) + std*norm.pdf(z)   # E[max(0, f - f_best)]

    UCB — the same idea with one explicit dial
    
        
    Upper Confidence Bound is an optimist's score: assume every point is as good as its mean plus a few standard deviations. The single knob κ sets the explore/exploit balance directly — small κ trusts the mean and exploits, large κ chases the widest bands. Both EI and UCB are cheap to evaluate, so we can maximise them over a dense grid.
    
        
    def ucb(mean, std, kappa=1.8):
    return mean + kappa*std      # small kappa -> exploit, large kappa -> explore

# an 'acquisition function' is a cheap proxy for 'where is the next real eval
# most likely to pay off'. Its ARGMAX is the single point we choose to sample.

    The loop: fit → maximise acquisition → evaluate → repeat
    
        
    This is the entire method. Fit the surrogate to everything seen so far, score a dense grid of candidates with the acquisition, pick its argmax, spend one real (expensive) evaluation there, append it, and go again. A dozen iterations is usually plenty — exactly the loop the LOOK tab runs on every click.
    
        
    Xs = np.linspace(*bounds, 400)                 # a fine candidate grid
for t in range(12):                            # a handful of expensive evals
    mean, std = gp_posterior(X, y, Xs)         # 1. fit surrogate to data so far
    acq       = expected_improvement(mean, std, y.max())   # 2. score candidates
    x_next    = Xs[np.argmax(acq)]             # 3. pick the acquisition's argmax
    y_next    = objective(x_next)              # 4. spend ONE real evaluation
    X = np.append(X, x_next); y = np.append(y, y_next)     # 5. repeat
print("best x:", X[np.argmax(y)], " best f:", y.max())

    The one-liner — and why it beats grid/random search
    
        
    scikit-optimize's gp_minimize (or BoTorch/Ax on the GPU) does all of the above and tunes the kernel too. Reach for BO when each evaluation is genuinely expensive and the budget is small — hyperparameter tuning, materials/drug screening, robotics, A/B tests. Grid and random search ignore everything they've learned; BO spends every evaluation where its uncertainty-aware model says the payoff is highest.
    
        
    from skopt import gp_minimize          # scikit-optimize; BoTorch/Ax for torch/GPU

# skopt MINIMIZES, so negate to maximize:
res = gp_minimize(lambda x: -objective(x[0]), [bounds],
                  n_calls=15, acq_func="EI")
print(-res.fun, res.x)                 # best value, best x  -- in ~15 evals

# grid/random search waste calls; BO spends each one where payoff is highest.
# cost: the GP is O(N^3) -> BO shines for EXPENSIVE f with a SMALL eval budget.
# high-dim / batch / parallel -> Thompson sampling, TuRBO, q-EI (BoTorch).
