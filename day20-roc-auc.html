📈 ROC & AUC — Scoring a Classifier Beyond Accuracy

Drag the threshold and watch the ROC point move — pull the classes apart and watch AUC climb

On the left a binary classifier has produced a score for every example: the rose hump is the true negatives, the sky hump is the true positives. A classifier never decides by itself — you pick a threshold (the white line). Everything to the right of it is called positive, everything to the left negative. Drag the threshold slider and we actually count the four outcomes — TP, FP, TN, FN — and turn them into a single dot: (FPR, TPR), which slides along the ROC curve on the right. The full ROC is traced by sweeping every threshold; the dashed diagonal is a random classifier. The AUC — area under that curve, summed with the trapezoid rule — is one number for the whole thing: 0.5 = random, 1.0 = perfect. Now drag the separation slider: as the two humps pull apart the scores separate cleanly, the ROC bows toward the top-left, and AUC marches toward 1.0; slide them back together and ROC collapses onto the diagonal at ~0.5. Every number here is computed live from the data — nothing is faked.

Top: the two score distributions (negatives in rose, positives in sky) with your threshold as the white line; everything right of it is predicted positive. Bottom: the full ROC curve swept over all thresholds, the dashed random diagonal, and the big white dot is the operating point at your current threshold. The shaded region under the curve is the AUC.

🔨 Build it yourself

    
Build a ROC curve and AUC from raw scores with no library doing the work for you: take a list of scores and true labels, sort by score, sweep every threshold, compute TPR and FPR at each, plot the (FPR, TPR) points to draw the curve, and integrate the area with the trapezoid rule to get the AUC. Then see the one-line scikit-learn call you'd actually use in practice.

    
    
      
  1. 
        
    1
    Start from scores and true labels
    
        
    

    # a classifier outputs a continuous SCORE per example (e.g. predicted
# probability of being positive), NOT a hard yes/no. the labels are the
# ground truth: 1 = positive, 0 = negative. ROC lives on these scores.
import numpy as np
rng = np.random.default_rng(0)
# 200 negatives centered low, 200 positives centered high (overlapping)
neg = rng.normal(0.0, 1.0, 200)
pos = rng.normal(2.0, 1.0, 200)            # separation = 2.0
scores = np.concatenate([neg, pos])
labels = np.concatenate([np.zeros(200), np.ones(200)]).astype(int)
    
          
        
    
      
    2. 
      
  2. 
        
    2
    Sort by score (descending)
    
        
    

    # every distinct score is a candidate threshold. sorting once, high to
# low, lets us sweep thresholds by walking the list top to bottom: as we
# pass each example we "flip" it from predicted-negative to positive.
order = np.argsort(-scores)        # indices, highest score first
s_sorted = scores[order]
y_sorted = labels[order]
P = labels.sum()                   # total actual positives
Nn = len(labels) - P               # total actual negatives
    
          
        
    
      
    3. 
      
  3. 
        
    3
    Compute the confusion matrix at one threshold
    
        
    

    # at a chosen threshold t, anything with score >= t is predicted
# positive. count the four cells. TPR and FPR are the two rates that
# the ROC plots — recall on the y-axis, false-alarm rate on the x-axis.
def confusion(t):
    pred = (scores >= t).astype(int)
    TP = int(((pred == 1) & (labels == 1)).sum())
    FP = int(((pred == 1) & (labels == 0)).sum())
    TN = int(((pred == 0) & (labels == 0)).sum())
    FN = int(((pred == 0) & (labels == 1)).sum())
    return TP, FP, TN, FN
TP, FP, TN, FN = confusion(1.0)
TPR = TP / (TP + FN)               # recall / sensitivity
FPR = FP / (FP + TN)               # 1 - specificity
    
          
        
    
      
    4. 
      
  4. 
        
    4
    Sweep every threshold → list of (FPR, TPR)
    
        
    

    # walk the score-sorted list. start at the strictest threshold (predict
# nothing positive) = point (0,0). each step lowers the threshold past
# one example: a true positive bumps TPR, a true negative bumps FPR.
tp = fp = 0
roc = [(0.0, 0.0)]
for yi in y_sorted:                # already sorted high->low score
    if yi == 1: tp += 1            # caught a real positive
    else:       fp += 1            # raised a false alarm
    roc.append((fp / Nn, tp / P))  # (FPR, TPR) after this threshold
# roc now ends at (1,1): the loosest threshold predicts all positive
fprs = [p[0] for p in roc]
tprs = [p[1] for p in roc]
    
          
        
    
      
    5. 
      
  5. 
        
    5
    Plot the ROC curve
    
        
    

    # plotting TPR against FPR draws the ROC. the dashed diagonal y = x is
# a coin-flip classifier; a good model bows up toward the top-left
# corner (high TPR while FPR is still low). top-left = perfect.
import matplotlib.pyplot as plt
plt.plot(fprs, tprs, "-", label="ROC")
plt.plot([0, 1], [0, 1], "--", label="random")   # the diagonal
plt.xlabel("FPR (1 - specificity)"); plt.ylabel("TPR (recall)")
plt.legend(); plt.show()
    
          
        
    
      
    6. 
      
  6. 
        
    6
    Integrate the area = AUC (trapezoid rule)
    
        
    

    # AUC is literally the area under that curve. since the points are
# already sorted by FPR, sum the area of each little trapezoid between
# consecutive points: width * average height. 0.5 = random, 1.0 = perfect.
auc = 0.0
for i in range(1, len(roc)):
    x0, y0 = roc[i - 1]
    x1, y1 = roc[i]
    auc += (x1 - x0) * (y0 + y1) / 2.0     # trapezoid area
print("AUC =", round(auc, 4))
# np.trapz does the same in one line: np.trapz(tprs, fprs)
    
          
        
    
      
    7. 
      
  7. 
        
    7
    Sanity check: AUC = P(score(pos) > score(neg))
    
        
    

    # beautiful identity: AUC equals the probability that a randomly chosen
# positive is scored higher than a randomly chosen negative. count all
# positive-vs-negative pairs the model ranks correctly. it should match.
wins = ties = 0
for sp in pos:
    for sn in neg:
        if   sp > sn: wins += 1
        elif sp == sn: ties += 1
auc_rank = (wins + 0.5 * ties) / (len(pos) * len(neg))
print("rank AUC =", round(auc_rank, 4), "  (matches the area)")

    In the real world: scikit-learn (and when to use PR instead)
    
        
    

    from sklearn.metrics import roc_curve, roc_auc_score, average_precision_score

# the curve points and the single-number AUC, both in one call each:
fpr, tpr, thr = roc_curve(labels, scores)
print("AUC =", roc_auc_score(labels, scores))     # threshold-independent

# when positives are RARE (fraud, disease), ROC can look optimistic
# because a huge TN count keeps FPR tiny. prefer the precision-recall
# curve / average precision there — it ignores TN and tracks the rare class.
print("avg precision =", average_precision_score(labels, scores))

🎉 Day 20 of MachineLearningFromZero done.
A classifier outputs a score, and a threshold turns it into a decision; the confusion matrix counts TP/FP/TN/FN, from which TPR (recall) and FPR (1 − specificity) fall out. Sweeping the threshold traces the ROC curve, the diagonal is random and the top-left is perfect, and AUC — the area under it — equals the chance a positive outscores a negative, summarizing the model independent of any single threshold and staying robust under class imbalance. When positives are rare, reach for the precision-recall curve instead. Next up: precision, recall & F1 — picking the operating point that matches the real cost of each mistake.
