🎰 Multi-Armed Bandits — When Every Pull Costs, How Do You Balance Exploring New Arms Against Exploiting the Best One?

Every arm hides a payout rate. Press "Pull" and watch three strategies each decide which lever to yank next — then watch their regret curves diverge. Day 38 spent an expensive evaluation where a surrogate said the payoff was highest. Today the choices are discrete: a row of slot machines, each paying out 1 with some hidden probability and 0 otherwise. You have a fixed number of pulls and want the most total reward. The catch is the same explore-vs-exploit tension — pull the arm that has looked best so far (exploit) and you might be stuck on an unlucky loser; pull an under-tried arm (explore) and you spend a turn that might have paid off. Three classic rules solve this and they all run live and for real below on the same bandit.

The scoreboard is regret: how much reward you gave up versus an oracle that always pulled the best arm, Regret(T) = T·μ* − Σ μaₜ. A good strategy makes regret grow sublinearly (it stops making mistakes); a bad one grows it in a straight line. ε-greedy exploits the best empirical mean but rolls a die with probability ε to explore at random — simple, but it keeps wasting that ε fraction forever. UCB1 pulls the arm with the highest optimistic bound μ̂ᵢ + c·√(ln t / nᵢ): the confidence bonus shrinks as an arm is pulled, so exploration decays on its own. Thompson sampling keeps a Beta(1+wins, 1+losses) belief per arm, draws one sample from each, and pulls the winner — probability-matching that is often the strongest of all. All three run on every pull; the panel shows the strategy you select, and the regret chart races all three.

🔨 Build it yourself

    
A bandit algorithm is astonishingly little code — no gradients, no matrices, just a running tally per arm and a rule for picking the next one. Here are the three classics in full, plus the loop that scores them on regret. Everything the LOOK tab does is in these snippets.

    1
    The bandit — N arms with hidden Bernoulli rates
    
        
    Each arm pays out 1 with some hidden probability p and 0 otherwise. You never see the ps — only the 0/1 outcomes of the pulls you spend. The best you could do is always pull the arm with the highest p; the whole game is finding it fast.
    
        
    import numpy as np
rng = np.random.default_rng(0)

true_rates = np.array([0.25, 0.45, 0.55, 0.72, 0.38])  # HIDDEN from the learner
K          = len(true_rates)
best_arm   = true_rates.argmax()          # the oracle's choice (unknown to us)
mu_star    = true_rates.max()

def pull(arm):
    return 1.0 if rng.random() < true_rates[arm] else 0.0   # one Bernoulli draw

    ε-greedy — exploit the best, explore at random
    
        
    Keep the empirical mean of each arm. Most of the time pull the arm with the highest mean (exploit). With probability ε, ignore all that and pull a uniformly random arm (explore). One knob, dead simple — but the constant ε means it keeps throwing away that fraction of pulls forever, so its regret grows linearly.
    
        
    def eps_greedy(counts, values, eps=0.1):
    if rng.random() < eps:                 # EXPLORE: random arm
        return rng.integers(K)
    return int(np.argmax(values))          # EXPLOIT: best empirical mean

# values[a] = running mean reward of arm a ; counts[a] = times pulled
# update after each pull:
#   counts[a] += 1
#   values[a] += (reward - values[a]) / counts[a]   # incremental mean

    UCB1 — optimism in the face of uncertainty
    
        
    Add a confidence bonus to each mean: μ̂ᵢ + c·√(ln t / nᵢ). The bonus is big for arms pulled few times (small nᵢ) and shrinks as you learn them; the ln t keeps a tiny curiosity alive so nothing is abandoned forever. Pull the arm with the highest bound. The weight c tunes exploration — Auer's original UCB1 fixes c = √2 ≈ 1.41; a smaller value (0.7 here) explores less and does better on short horizons — and it is provably O(ln T) regret.
    
        
    def ucb1(counts, values, t, c=0.7):
    for a in range(K):
        if counts[a] == 0:                     # play each arm once first
            return a
    bonus = c * np.sqrt(np.log(t) / counts)    # shrinks as an arm is pulled
    return int(np.argmax(values + bonus))      # optimistic index (c=sqrt(2)=UCB1)

# small n  -> big bonus  -> "worth checking"
# big  n   -> tiny bonus -> judged on its mean alone

    Thompson sampling — pull in proportion to being best
    
        
    Be Bayesian. For Bernoulli arms the conjugate belief is a Beta(1+wins, 1+losses) posterior per arm — flat at the start, sharpening toward the true rate with every pull. Each round, draw one sample from every arm's posterior and pull whichever sample is highest. Arms that are plausibly the best get pulled often; arms shown to be poor rarely resurface. Tiny code, frequently the best empirical performer.
    
        
    def thompson(wins, losses):
    # sample a plausible rate from each arm's Beta posterior...
    samples = rng.beta(1 + wins, 1 + losses)
    return int(np.argmax(samples))         # ...and pull the winner

# update after each pull:
#   wins[a]   += reward          # a success bumps alpha
#   losses[a] += (1 - reward)    # a failure bumps beta
# uncertainty (wide Beta) => this arm sometimes wins the draw => gets explored

    The loop — play, learn, and score on regret
    
        
    Every strategy is the same loop: pick an arm, pull it, update that arm's tally. To compare them fairly, track regret — how much expected reward each pull gave up versus always pulling the best arm. A good strategy's regret curve bends over and flattens; a bad one's keeps climbing.
    
        
    counts, values = np.zeros(K), np.zeros(K)
regret = 0.0; curve = []
for t in range(1, 2001):
    a = ucb1(counts, values, t)            # or eps_greedy / thompson
    r = pull(a)                            # spend one pull
    counts[a] += 1
    values[a] += (r - values[a]) / counts[a]
    regret += mu_star - true_rates[a]      # gap vs the oracle's arm
    curve.append(regret)
print("regret:", regret, " %best:", counts[best_arm] / counts.sum())

    Why UCB/Thompson win — and where bandits go next
    
        
    ε-greedy explores blindly and forever; UCB1 and Thompson explore where the uncertainty is and taper off as they learn, so their regret is logarithmic instead of linear. Bandits power A/B testing, ad and content selection, recommendations, and clinical trials — anywhere you must earn while you learn. Add a state that your actions change and delayed rewards, and the bandit grows up into full reinforcement learning.

🎉 Day 39 of MachineLearningFromZero done.
You turned Day 38's explore/exploit tension into a discrete decision problem. A row of arms hid Bernoulli payout rates; the score was regret T·μ* − Σ μaₜ, the reward left on the table versus an oracle. ε-greedy exploited the best mean but explored blindly at rate ε — linear regret. UCB1 added an optimistic confidence bonus c·√(ln t / nᵢ) that decays as arms are learned, and Thompson sampling kept a Beta(1+wins,1+losses) belief and pulled in proportion to being best — both bend regret down to logarithmic and beat greedy handily. That is the whole game: explore just enough, then commit. 👉 Tomorrow — Day 40: Q-Learning (Reinforcement Learning) — when your actions move you between states and rewards arrive with a delay, the bandit grows a memory: value iteration and temporal-difference learning teach an agent to plan, not just to pick.
