⚡ XGBoost Internals — What Turns Plain Gradient Boosting Into the Algorithm That Keeps Winning: a Regularized Objective, Second-Order Taylor Splits, and γ-Pruned Trees

Day 15 boosted weak trees by fitting each new one to the previous model's residuals. XGBoost keeps that skeleton but rebuilds the engine — and every piece of the rebuild is on screen below, computed for real on a tiny 1-D dataset. Instead of chasing raw residuals, XGBoost writes down a single regularized objective, approximates it with a second-order Taylor expansion so every sample carries both a gradient gᵢ and a curvature hᵢ, and then derives — in closed form — the best leaf weight and the exact gain of any candidate split. Two knobs it adds, λ and γ, decide how hard leaf weights shrink and whether a split is even worth making.

Below is a single tree being grown the XGBoost way. Each point gets its gᵢ (gradient of the loss at the current prediction) and hᵢ (its second derivative). Group sums G=Σgᵢ and H=Σhᵢ are all a leaf needs: its optimal weight is w* = −G/(H+λ) and the value of a split is Gain = ½[ GL²/(HL+λ) + GR²/(HR+λ) − G²/(H+λ) ] − γ. Sweep the threshold and the gain curve (top-right) rises and falls; the dashed γ line is the bar a split must clear — below it, XGBoost refuses to split and the branch stays a leaf. Drag λ and watch the leaf weights shrink; drag γ and watch splits appear and vanish. The amber overlay shows what plain gradient boosting would have done: divide by the raw count n instead of H+λ, and never prune.

🔨 Build it yourself

    
XGBoost's inner loop is startlingly small once you have gᵢ and hᵢ: a leaf is two sums, a split is one formula, and a tree is that formula applied greedily. Here is the whole thing — the second-order setup, the gain, the split search, the closed-form leaf weight, the boosting round with its learning rate, and finally the two systems tricks (histogram binning and sparsity-aware default directions) that make it fast. Everything the LOOK tab computes is in these snippets.

    
    
      
  1. 
        
    1
    g and h — the second-order Taylor expansion of the loss
    
        
    We are adding one more tree to an ensemble whose current raw prediction for point i is Fᵢ. Rather than fit the residual, XGBoost Taylor-expands the loss around Fᵢ to second order: it needs the gradient gᵢ = ∂l/∂F and the Hessian hᵢ = ∂²l/∂F². For squared error hᵢ=1 always; for log-loss hᵢ=pᵢ(1−pᵢ) — the curvature down-weights points the model is already confident about.
    
        
    import numpy as np

def grad_hess(y, F, loss="logloss"):
    if loss == "mse":                 # l = 1/2 (y - F)^2
        g = F - y                     # dl/dF
        h = np.ones_like(F)           # d2l/dF2  ==  1  (constant!)
    else:                             # logistic:  l = log(1+e^F) - yF
        p = 1.0 / (1.0 + np.exp(-F))  # current probability
        g = p - y                     # dl/dF
        h = p * (1.0 - p)             # d2l/dF2  ==  p(1-p)  (varies per point)
    return g, h

    The leaf weight and the split gain — both in closed form
    
        
    Plug the Taylor approximation into the objective with an L2 penalty ½λΣw² and the optimum for a leaf holding index set I is exactly w* = −G/(H+λ) with G=Σgᵢ, H=Σhᵢ. Its contribution to the objective (the "structure score") is −½·G²/(H+λ). A split's gain is how much that score improves, minus a per-leaf cost γ.
    
        
    def leaf_weight(G, H, lam):
    return -G / (H + lam)                 # closed-form optimum, L2-shrunk

def split_gain(GL, HL, GR, HR, lam, gamma):
    G, H = GL + GR, HL + HR
    sim  = lambda g, h: g*g / (h + lam)    # "similarity score" of a node
    return 0.5 * (sim(GL, HL) + sim(GR, HR) - sim(G, H)) - gamma
# gain is ALWAYS >= 0 before subtracting gamma; gamma is the bar it must clear

    Exact split finding — sweep every threshold
    
        
    Sort the points by the feature, then walk left to right accumulating GL, HL; the right side is just the totals minus the left. Score each threshold with the gain formula and keep the best. This is the "exact greedy" algorithm — O(#points) per feature after the sort.
    
        
    def best_split(g, h, x, lam, gamma):
    order = np.argsort(x)
    g, h, x = g[order], h[order], x[order]
    G, H = g.sum(), h.sum()
    GL = HL = 0.0
    best = (0.0, None)                      # (gain, threshold)
    for i in range(len(x) - 1):
        GL += g[i]; HL += h[i]              # move point i to the left child
        if x[i] == x[i+1]:                  # can't split between equal values
            continue
        gain = split_gain(GL, HL, G-GL, H-HL, lam, gamma)
        if gain > best[0]:
            best = (gain, 0.5*(x[i] + x[i+1]))
    return best                             # best[0] <= 0  =>  don't split

    Grow the tree — split while it pays, else make a leaf
    
        
    Recurse: find the best split; if its gain is positive (it cleared γ) and we are under the depth limit, split and recurse on both children; otherwise stop and emit a leaf with weight w* = −G/(H+λ). That single test — gain > 0 — is where γ does its pruning.
    
        
    def build(idx, g, h, x, depth, lam, gamma, max_depth=3):
    G, H = g[idx].sum(), h[idx].sum()
    gain, thr = best_split(g[idx], h[idx], x[idx], lam, gamma)
    if depth >= max_depth or gain <= 0 or thr is None:
        return {"leaf": leaf_weight(G, H, lam)}    # gamma pruned it -> leaf
    left  = idx[x[idx] <  thr]
    right = idx[x[idx] >= thr]
    return {"thr": thr,
            "L": build(left,  g, h, x, depth+1, lam, gamma, max_depth),
            "R": build(right, g, h, x, depth+1, lam, gamma, max_depth)}

    The boosting round — and how plain GB differs
    
        
    Each round: compute g, h at the current predictions, grow one tree, and add its leaf outputs to F scaled by a learning rate η (shrinkage — yet another regularizer). Plain gradient boosting is the same loop with h≡1, λ=γ=0, so its leaf is the mean residual −G/n and it never prunes.

for m in range(n_rounds):
    g, h = grad_hess(y, F, loss)            # <-- 2nd order:  plain GB uses h = 1

    F += eta * predict(tree, X)             # eta = learning-rate shrinkage

# plain-GB leaf: w  = -G / n                # 1st order, no lambda, no gamma
#   -> identical ONLY for squared error with lam = gamma = 0 (there H == n)

    The systems tricks — histogram binning & sparsity-aware splits
    
        
    Exact split finding sorts every feature every node. Real XGBoost instead bins each feature into ~256 buckets once, then a split search is a single pass summing G,H per bin — turning a sort into a histogram (this is the hist / approx tree method). And it is sparsity-aware: each node learns a default direction for missing values by trying both and keeping whichever gives more gain, so zeros and NaNs cost almost nothing.
    
        
    # histogram: bin once, then split search is a bin-wise cumulative sum
bins = np.digitize(x, bin_edges)             # ~256 buckets, computed ONCE
Gb = np.bincount(bins, weights=g)            # per-bin gradient sums
Hb = np.bincount(bins, weights=h)            # per-bin hessian  sums
# gain over thresholds = cumulative Gb/Hb from the left -- no re-sorting

# sparsity-aware: send missing rows the way that maximises gain

    gain = gain_with_missing_sent(direction)   # try both once
best_default = argmax over direction           # store it in the node

You opened up the engine behind the algorithm that keeps winning tabular competitions. XGBoost keeps gradient boosting's shape but writes a regularized objective and approximates it with a second-order Taylor expansion, so every point carries a gradient gᵢ and a curvature hᵢ. From those, a leaf's best weight is closed-form w* = −G/(H+λ) and a split's value is ½[GL²/(HL+λ)+GR²/(HR+λ)−G²/(H+λ)] − γ. λ shrinks leaf weights; γ is the bar a split must clear or it is pruned to a leaf. For squared error hᵢ=1 and it collapses back to plain variance-reduction boosting — the Hessian only earns its keep on losses like log-loss where hᵢ=pᵢ(1−pᵢ).
