🫧 Gaussian Mixtures & EM — Soft Clustering

Watch ellipses grow to fit the data — and points blend colours where clusters overlap.

K-Means forces every point into exactly one round cluster. A Gaussian Mixture Model is softer and smarter: it models the data as a blend of bell-curves (Gaussians) that can be stretched and tilted into ellipses, and it gives every point a probability of belonging to each cluster. It's fit with the EM algorithm — alternating an E-step (how responsible is each cluster for each point?) and an M-step (move each Gaussian to fit the points it's responsible for). 🔨 Build it yourself

    
A 2D Gaussian pdf, an E-step that computes responsibilities, an M-step that updates means/covariances/weights, and a loop that repeats until the log-likelihood stops rising. ~90 lines of vanilla JS.

    
    
      
  1. 
        
    1
    The 2D Gaussian pdf
    
        
    Each cluster is a 2D Gaussian with a mean (centre) and a 2×2 covariance matrix (shape). The pdf needs the covariance's determinant and inverse — cheap closed forms for a 2×2. This value is "how likely is this point under this Gaussian".
    
        
    function gauss(p, mean, cov){
  const [a,b,c,d] = cov;                 // [[a,b],[c,d]]
  const det = a*d - b*c;
  const inv = [ d/det, -b/det, -c/det, a/det ];
  const dx = p.x-mean.x, dy = p.y-mean.y;
  const m  = dx*(inv[0]*dx+inv[1]*dy) + dy*(inv[2]*dx+inv[3]*dy);
  return Math.exp(-0.5*m) / (2*Math.PI*Math.sqrt(det));
}
    
      
    2. 
      
  2. 
        
    2
    E-step: responsibilities
    
        
    For each point, compute weight × pdf for every cluster, then normalise so they sum to 1. Those normalised numbers are the responsibilities — the soft, probabilistic assignment of the point across clusters. This is the "expectation" in Expectation-Maximization.
    
        
    function eStep(points, model){
  return points.map(p => {
    const r = model.map(k => k.weight * gauss(p, k.mean, k.cov));
    const s = r.reduce((a,b)=>a+b, 0) || 1e-9;
    return r.map(v => v/s);          // responsibilities, sum to 1
  });
}

    3
    M-step: update the Gaussians
    
        
    Now re-fit each Gaussian using the responsibilities as soft counts. The new mean is a responsibility-weighted average of the points; the new covariance is the weighted spread around that mean; the new weight is the cluster's share of total responsibility. This is the "maximization" — it maximises the likelihood given the current soft assignments.
    
        
    function mStep(points, resp, K){
  return Array.from({length:K}, (_, k) => {
    const Nk = resp.reduce((a,r)=>a+r[k], 0) || 1e-9;
    const mx = resp.reduce((a,r,i)=>a+r[k]*points[i].x,0)/Nk;
    const my = resp.reduce((a,r,i)=>a+r[k]*points[i].y,0)/Nk;
    let a=0,b=0,d=0;
    points.forEach((p,i)=>{ const w=resp[i][k], dx=p.x-mx, dy=p.y-my;
      a+=w*dx*dx; b+=w*dx*dy; d+=w*dy*dy; });
    return { mean:{x:mx,y:my}, cov:[a/Nk,b/Nk,b/Nk,d/Nk], weight:Nk/points.length };
  });
}

  4. 
        
    4
    Loop until log-likelihood converges
    
        
    Alternate E and M. After each round, the total log-likelihood (sum of log of the mixture density at every point) never decreases — so stop when its improvement falls below a small threshold. That monotonic climb is EM's guarantee and a great convergence check.
    
        
    let prev = -Infinity;
for (let it = 0; it < 100; it++){
  const resp  = eStep(points, model);
  model = mStep(points, resp, K);
  const ll = logLikelihood(points, model);
  if (ll - prev < 1e-4) break;   // converged
  prev = ll;
}
