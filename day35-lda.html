🧭 Linear Discriminant Analysis — Project to the Axis That Separates the Classes

These two clouds are stretched the wrong way. PCA chases the direction of biggest spread and points straight along the classes — a projection where they blur together. LDA uses the labels to find the one axis that pulls the classes apart. Drag the sliders to move the classes and watch both axes — and the decision threshold — recompute live.

PCA is unsupervised: it maximises total variance and never looks at the labels, so when each class is elongated it happily picks the long within-class direction — useless for telling the classes apart. LDA is supervised: it maximises the Fisher criterion J(w) = (wᵀ S_b w) / (wᵀ S_w w) — between-class scatter S_b over within-class scatter S_w — i.e. it looks for the projection where the class means are far apart relative to the spread inside each class. For two classes that has the tidy closed form w ∝ S_w⁻¹(μ₁ − μ₀). Every number below — the class means, S_w, S_b, the LDA axis, the PCA axis and the accuracy of classifying along each — is solved for real in your browser, no libraries.

🔨 Build it yourself

    
scikit-learn ships LDA as LinearDiscriminantAnalysis — it is both a classifier and a supervised dimensionality reducer. Below is the whole workflow: fit it, use it to project down to the class-separating axes, then the from-scratch scatter matrices and eigen-solve that produced every axis you just dragged.

    
    
      
  1. 
        
    1
    Fit LDA as a classifier
    
        
    Out of the box LDA is a fast, closed-form classifier. It assumes each class is Gaussian with a shared covariance, which makes its decision boundaries linear — exactly the green line in the demo.
    
        
    from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline

clf = make_pipeline(StandardScaler(), LinearDiscriminantAnalysis())
clf.fit(X_train, y_train)
print("test accuracy:", clf.score(X_test, y_test))
    
      
    2. 
      
  2. 
        
    2
    Use it as a supervised reducer — and compare with PCA
    
        
    transform projects your data onto the class-separating axes (at most C − 1 of them for C classes). This is the supervised cousin of PCA: same "reduce to a few axes" idea, but LDA uses the labels to keep the axes that discriminate instead of the axes that merely have the most variance.
    
        
    from sklearn.decomposition import PCA
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis

X_lda = LinearDiscriminantAnalysis(n_components=1).fit_transform(X, y)  # uses y
X_pca = PCA(n_components=1).fit_transform(X)                            # ignores y

# On elongated classes X_lda separates the labels; X_pca often does not.
    
      
    3. 
      
  3. 
        
    3
    From scratch: class means and within-class scatter S_w
    
        
    S_w adds up how spread each class is around its own mean. It is the sum of the per-class scatter matrices — the "noise" LDA wants in the denominator, so it can divide it out.
    
        
    import numpy as np

classes = np.unique(y)
means   = {c: X[y == c].mean(axis=0) for c in classes}
overall = X.mean(axis=0)
d = X.shape[1]

Sw = np.zeros((d, d))                       # within-class scatter
for c in classes:
    Xc = X[y == c] - means[c]
    Sw += Xc.T @ Xc                         # sum of (x-mu_c)(x-mu_c)^T
    
      
    4. 
      
  4. 
        
    4
    From scratch: between-class scatter S_b
    
        
    S_b measures how far the class means sit from the grand mean, weighted by class size — the "signal" LDA wants in the numerator. Maximising signal-over-noise is the whole idea.
    
        
    Sb = np.zeros((d, d))                       # between-class scatter
for c in classes:
    n_c  = (y == c).sum()
    diff = (means[c] - overall).reshape(d, 1)
    Sb  += n_c * (diff @ diff.T)            # n_c (mu_c - mu)(mu_c - mu)^T
    
      
    5. 
      
  5. 
        
    5
    Solve for the axes — the exact demo engine
    
        
    Maximising the Fisher criterion J(w) = (wᵀS_b w)/(wᵀS_w w) is a generalized eigenvalue problem: the best axes are the top eigenvectors of S_w⁻¹ S_b. For two classes that collapses to the one-liner w ∝ S_w⁻¹(μ₁ − μ₀) — the closed form the demo uses. This is what runs in the page.
    
        
    import numpy as np

# --- two-class closed form (what the LOOK tab draws) ---
w2 = np.linalg.inv(Sw) @ (means[1] - means[0])
w2 /= np.linalg.norm(w2)

# --- general multi-class: top eigenvectors of Sw^-1 Sb ---
eigvals, eigvecs = np.linalg.eig(np.linalg.inv(Sw) @ Sb)
order = np.argsort(eigvals.real)[::-1]        # largest first
W = eigvecs[:, order[:n_components]].real      # the LDA axes

X_proj = X @ W                                 # project onto them
    
      
    6. 
      
  6. 
        
    6
    Project, threshold, and know the assumptions
    
        
    Classify a point by projecting it and comparing to the midpoint of the projected class means (equal priors) — the dashed threshold in the demo. LDA is fast, needs no tuning, and regularises naturally when data is scarce. But it assumes Gaussian classes with a shared covariance: if the classes have very different shapes, reach for QDA; if you only want variance-preserving compression with no labels, that's PCA.
    
        
    # project + nearest-projected-mean rule (= midpoint threshold for 2 classes)
proj_means = {c: means[c] @ w2 for c in classes}
def predict(x):
    t = x @ w2
    return min(classes, key=lambda c: abs(t - proj_means[c]))

# when to use what:
# LDA : Gaussian classes, shared covariance -> linear boundary, supervised
# QDA : per-class covariance              -> curved boundary
# PCA : no labels, keep variance          -> unsupervised compression
# Singular Sw (few samples)? use shrinkage: LinearDiscriminantAnalysis(
#     solver="lsqr", shrinkage="auto")

🎉 Day 35 of MachineLearningFromZero done.
You watched LDA and PCA fight over the same 2-D cloud. PCA, blind to the labels, chased raw variance and pointed along the elongated classes — a projection where they blur into one hump. LDA used the labels to maximise the Fisher criterion J(w)=(wᵀS_bw)/(wᵀS_ww): between-class scatter S_b over within-class scatter S_w. You saw the two-class closed form w ∝ S_w⁻¹(μ₁−μ₀), the general solve as the top eigenvectors of S_w⁻¹S_b, the projection collapsing the points onto that axis, and a midpoint decision threshold — with LDA's accuracy leaving PCA's far behind whenever spread and separation disagree. 👉 Tomorrow — Day 36: Bayesian Linear Regression — put a distribution over the weights and get calibrated uncertainty on every prediction, not just a point estimate.
